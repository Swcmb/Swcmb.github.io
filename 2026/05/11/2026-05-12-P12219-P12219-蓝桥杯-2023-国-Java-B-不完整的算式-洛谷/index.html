[洛谷 P12219] 不完整的算式详解

1. 标题

洛谷 P12219 [蓝桥杯 2023 国 Java B] 不完整的算式详解

2. 题意简述

输入一个不完整的算式，其中包含两个整数和一个运算符，其中一个数字可能用 ? 表示。需要根据给定的算式和运算符，计算出缺失的数字，或者判断运算符。

3. 思路分析

朴素解法

最直观的做法是尝试所有可能的数字组合，然后检查它们是否满足给定的算式。这种方法的时间复杂度是 $O(n^2)$，其中 $n$ 是数字的可能值范围，这显然是低效的。

优化方向

我们可以通过分析算式的结构来减少需要尝试的组合数量。例如，如果 ? 在等号右边，我们可以直接计算等号左边的表达式。

最终解法

  • 状态定义：我们将问题分为三种情况：缺失的数字是 A、B 或 C。
  • 转移方程：
    • 当缺失的数字是 A 时，我们有以下逆运算公式：
      • A + B = C → A = C - B
      • A - B = C → A = C + B
      • A * B = C → A = C // B（整除）
      • A / B = C → A = C * B
    • 当缺失的数字是 B 时，我们有以下逆运算公式：
      • A + B = C → B = C - A
      • A - B = C → B = A - C
      • A * B = C → B = C // A（唯一解保证整除）
      • A / B = C → B = A // C（唯一解保证整除）
    • 当缺失的数字是 C 时，我们需要检查所有可能的运算符，然后计算结果。

代码说明

```python

以下是修正后的代码

s = input().strip()

找到等号位置

eq_idx = s.index(‘=’)
left = s[:eq_idx] # 左边 “A op B”
right = s[eq_idx+1:] # 右边 “C”

判断问号的位置：右边、左边运算符、左边数字

if ‘?’ in right:

# 缺失的是 C，计算 A op B
# 先解析 left 中的 A, op, B
# 找到运算符的位置（第一个 + - * /）
for i, ch in enumerate(left):
    if ch in '+-*/':
        op = ch
        a_str = left[:i]
        b_str = left[i+1:]
        break
else:
    # 如果没有运算符，说明运算符就是 '?'
    pass
a = int(a_str)
b = int(b_str)
if op == '+':
    ans = a + b
elif op == '-':
    ans = a - b
elif op == '*':
    ans = a * b
else:  # '/'
    ans = a // b   # 整除
print(ans)

elif ‘?’ in left:

# 问号在左边，可能是 A、op 或 B
# 情况1：左边含有运算符 + - * / 且不是问号，则问号在 A 或 B 中
has_op = False
for ch in left:
    if ch in '+-*/':
        has_op = True
        op = ch
        break
if has_op:
    # 运算符已知，问号在 A 或 B 中
    pos_op = left.index(op)  # 运算符位置
    a_str = left[:pos_op]
    b_str = left[pos_op+1:]
    c = int(right)
    if '?' in a_str:
        # 缺失 A
        b = int(b_str)
        if op == '+':
            ans = c - b
        elif op == '-':
            ans = c + b   # A - B = C  => A = C + B
        elif op == '*':
            ans = c // b   # 整除（唯一解保证整除）
        else:  # '/'
            ans = c * b    # A / B = C  => A = C * B
        print(ans)
    else:  # '?' in b_str
        a = int(a_str)
        if op == '+':
            ans = c - a   # A + B = C  => B = C - A
        elif op == '-':
            ans = a - c   # A - B = C  => B = A - C
        elif op == '*':
            ans = c // a   # 整除（唯一解保证整除）
        else:  # '/'
            ans = a // c   # A / B = C  => B = A // C
        print(ans)
else:
    # 左边没有运算符，说明运算符就是 '?'，此时 left 形如 "数字?数字"
    # 解析 A 和 B（都是数字）
    q_idx = left.index('?')
    a_str = left[:q_idx]
    b_str = left[q_idx+1:]
    a = int(a_str)
    b = int(b_str)
    c = int(right)
    # 尝试四种运算符，找到使等式成立的那个
    if a + b == c:
        print('+')
    elif a - b == c:
        print('-')
    elif a * b == c:
        print('*')
    elif b != 0 and a // b == c:  # 整除，注意 b 不能为 0
        print('/')

else:

# 理论上不可能，因为输入保证有一个 '?'
pass

4. 复杂度分析

  • 时间复杂度：$O(1)$，因为代码中没有任何循环结构。
  • 空间复杂度：$O(1)$，因为代码中使用的变量数量不随输入规模变化。

6. 输入输出示例

输入：
10+?=?+?
输出：
0
解释：
等式左边是 10+?=?+?，我们可以看到运算符是 +，所以缺失的数字是 C。我们可以计算 10+0=10，所以 C=0。

7. 易错点 & 收获

  • 易错点：
    • 忘记处理除数为零的情况。
    • 输入解析错误，导致无法正确处理多位数。
  • 收获：
    • 学习了如何通过分析算式结构来优化算法。
    • 学习了逆运算公式的应用。
