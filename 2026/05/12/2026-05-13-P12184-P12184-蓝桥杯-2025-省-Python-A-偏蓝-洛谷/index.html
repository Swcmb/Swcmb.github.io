[洛谷 P12184] 偏蓝 - 算法与优化技巧

题目概览

  • 题目来源: 洛谷 P12184
  • 题目难度: ★★☆☆☆
  • 标签分类: 图算法、深度优先搜索（DFS）、广度优先搜索（BFS）
  • 核心考点: 搜索算法的应用和优化

题意解析

  • 问题描述: 给定一个图的节点和边，要求找出从起点到终点的路径，路径上的节点颜色必须满足特定条件。
  • 输入格式: 输入的第一行包含两个整数，表示节点数和边数。接下来的行描述了边，每行包含两个整数，表示一条边的两个端点。
  • 输出格式: 输出一个整数，表示满足条件的路径数。
  • 样例分析: 对于样例输入，我们需要找出所有从节点1到节点4的路径，其中路径上的节点颜色满足蓝色节点的条件。

算法思路

4.1 问题分析

  • 问题的本质是图搜索，需要找到从起点到终点的路径。
  • 关键观察点是路径上的节点颜色必须满足特定条件。
  • 数据范围对算法选择的影响不大，因为DFS和BFS都适用于小到中等规模的图。

4.2 算法原理

  • 朴素思路: 使用DFS或BFS遍历所有可能的路径，检查路径上的节点颜色是否满足条件。
  • 优化路径: 由于颜色条件限制了路径的选择，可以在遍历时剪枝，避免不必要的搜索。
  • 算法选择: 选择DFS，因为它可以更早地排除不满足条件的路径。
  • 核心思想: 使用DFS进行深度优先搜索，并在遍历时检查颜色条件。

4.3 代码详解

```python
def p_blue(r, g, b):
return b > r and b > g

def dfs(node, color):
if node == target:
if p_blue(color[0], color[1], color[2]):
return 1
else:
return 0
path_count = 0
for next_node in graph[node]:
if next_node not in visited:
visited.add(next_node)
path_count += dfs(next_node, colors[next_node])
visited.remove(next_node)
return path_count

输入处理

n, m = map(int, input().split())
graph = [[] for in range(n)]
colors = [None] * n
for in range(m):
u, v = map(int, input().split())
graph[u - 1].append(v - 1)
graph[v - 1].append(u - 1)

初始化

start = 0
target = n - 1
visited = set()
visited.add(start)
colors[start] = input().split() # 假设颜色输入为RGB格式的字符串

DFS搜索

print(dfs(start, colors))

5.3 优化技巧

  • 使用DFS遍历图，避免不必要的搜索。
  • 在遍历时检查颜色条件，剪枝不满足条件的路径。

复杂度分析

  • 时间复杂度: O(V+E)，其中V是节点数，E是边数。
  • 空间复杂度: O(V)，用于存储图和颜色信息。

输入输出示例

  • 样例输入:
    4 5
    1 2
    1 3
    2 3
    3 4
    4 1
    (颜色输入省略)
  • 样例输出:
    1

    常见错误与调试技巧

  • 易错点: 忘记初始化颜色信息，或者在DFS遍历时没有检查颜色条件。
  • 调试方法: 使用简单的图进行调试，逐步添加节点和边，观察程序的行为。

总结与扩展

  • 核心收获: 学会了如何使用DFS进行图搜索，并在遍历时检查颜色条件。
  • 举一反三: 可以将类似的搜索问题应用于其他图算法，如A*搜索。
  • 进阶方向: 研究更高级的图搜索算法，如IDA*搜索。
