标题

[蓝桥杯 2025 省 Java B] 逃离高塔 - 洛谷

题目概览

  • 题目来源: 洛谷
  • 题目难度: ★★★☆☆
  • 标签分类: 算法, 数据结构
  • 核心考点: 动态规划、前缀和优化

题意解析

  • 问题描述: 在一个三维空间中，有一个高塔，从塔顶向下看，可以看到一个由数字组成的图形。每个数字代表塔的一层高度。我们需要找到一种方法，通过移动塔的底部，使得所有数字的总和达到最小。
  • 输入格式: 整数数组 a，表示每层的高度。
  • 输出格式: 整数，表示最小总和。
  • 样例分析: 例如，对于输入 [1, 2, 3, 4]，我们可以通过将塔的底部向上移动一层来得到总和为 6。

算法思路（核心部分）

4.1 问题分析

  • 问题的本质是动态规划中的“最短路径”问题，即在给定条件下，如何通过最少的操作使总和最小。
  • 关键观察点：每层的高度之和与下一层的关系。
  • 数据范围对算法选择的影响：由于高度可能非常大，简单的累加或遍历可能不适用。

4.2 算法原理

  • 朴素思路: 直接遍历所有可能的移动方式，计算总和。
  • 优化路径: 使用动态规划的方法，记录已访问过的层的高度和对应的总和，避免重复计算。
  • 算法选择: 动态规划因其时间复杂度较低，适合处理此类问题。
  • 核心思想: 利用动态规划的思想，通过构建状态转移方程，逐步求解。

4.3 数学推导（如适用）

  • 状态转移方程：设当前层的总和为 S[i]，则 S[i] = S[i-1] + a[i]。
  • 最优解：当且仅当 S[i] 小于等于 S[j] 时，a[i] 应被移动到 j 的位置。
  • 复杂度分析：时间复杂度为 O(n^2)，空间复杂度为 O(n)。

4.4 多种解法对比（如适用）

解法 时间复杂度 空间复杂度 适用场景 代码难度
解法1 O(n^2) O(n) 小规模数据 简单
解法2 O(n log n) O(n) 中等规模 中等

代码详解

```python
def min_sum(a):
n = len(a)
if n == 0: return 0
dp = [0] * (n + 1)
dp[0] = a[0]
for i in range(1, n + 1):
dp[i] = min(dp[i], dp[i - 1] + a[i])
return dp[n]

5.1 核心代码段

  • dp = [0] * (n + 1): 初始化动态规划数组，用于存储每一层的最小总和。
  • dp[0] = a[0]: 第一层的高度就是其本身。
  • for i in range(1, n + 1):: 遍历每一层的移动。
  • dp[i] = min(dp[i], dp[i - 1] + a[i]): 根据前一层的最小总和，更新当前层的最小总和。
  • return dp[n]: 返回最终的最小总和。
