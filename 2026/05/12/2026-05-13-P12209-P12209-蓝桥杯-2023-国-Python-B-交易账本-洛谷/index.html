[洛谷 P12209] 交易账本 - 洛谷

题目概览

  • 题目来源: 洛谷
  • 题目难度: ★★★★☆
  • 标签分类: 算法, 数据结构
  • 核心考点: 交易账本的合法性检查

题意解析

在本题中，我们需要模拟一个交易账本，该账本记录了一系列的交易。每个交易包括多个输出项，每个输出项都有一个账户ID和一个金额。我们需要判断这些交易是否合法，即每个交易的输出金额是否等于其输入金额的总和，并且每个输出项是否已经被使用过。

输入格式:

  • 第一行是一个整数 ( t )，表示交易的数量。
  • 每个交易由两部分组成：
    • 第一部分是一个整数 ( n )，表示该交易的输出项数量。
    • 第二部分是两个整数 ( m ) 和 ( m+1 )，分别表示输出项的数量和下一个交易的索引。
  • 接下来的 ( 2n ) 行每行包含两个整数 ( s ) 和 ( e )，表示第 ( i ) 个交易的起始和结束索引。
  • 最后一行包含一个整数 ( k )，表示需要检查的账户ID的数量。

输出格式:

  • 如果所有交易都合法，输出 “YES”。
  • 如果存在不合法的交易，输出 “NO”。

算法思路

4.1 问题分析

  • 问题本质: 这是一个典型的合法性检查问题，涉及到对交易数据的遍历和验证。
  • 关键观察点:
    • 每个交易的输出项数量和起始索引。
    • 每个输出项的账户ID和金额。
    • 需要确保每个输出项的金额等于其输入金额的总和，并且每个输出项未被使用过。

4.2 算法原理

  • 朴素思路:
    • 遍历每个交易，检查其合法性。
    • 对于每个交易，遍历其输出项，检查金额是否匹配。
    • 使用一个二维数组来记录每个输出项的使用情况。
  • 优化路径:
    • 使用滚动数组来减少空间复杂度。
    • 预处理输入数据，提前计算一些中间结果以减少重复计算。
  • 算法选择:
    • 选择这种基于遍历和验证的方法，因为它能够有效地解决这个问题。
    • 对比其他算法，如动态规划或图论方法，它们在这个问题中不太适用，因为问题的本质不是最优化或寻找最大值/最小值。

4.3 数学推导

  • 在这个问题中，数学推导不是必需的，因为主要问题是逻辑验证，而不是需要通过数学公式来解决的问题。

    • 忽略了指针的正确移动。
    • 没有正确处理特殊交易的情况。
    • 没有正确更新总输入金额和总输出金额。

    • 使用打印语句跟踪指针的位置和变量的值。
