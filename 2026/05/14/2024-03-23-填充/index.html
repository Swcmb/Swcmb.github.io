问题描述

题目来源：蓝桥杯练习系统。

给定一个长度为 $n$ 的 01 字符串，其中部分位置标记为 ?。这些位置可以任意填充 0 或 1。请问如何填充这些位置，使得该 01 字符串中出现互不重叠的 00 和 11 子串数量最多？请输出这个最大数量。

输入输出格式

输入格式

输入一行，包含一个由 0、1 和 ? 组成的字符串。

输出格式

输出一行，包含一个整数，表示能够组成的互不重叠的 00 或 11 子串的最大个数。

样例

样例输入

```text
1110?0

样例输出

```text
2

样例解析

对于样例输入 1110?0，我们可以尝试不同的填充策略：

  • 若将 ? 填为 1，字符串变为 111010，最多包含 1 个 11 子串。
  • 若将 ? 填为 0，字符串变为 111000。
    • 可以找到 11（索引 0-1）和 00（索引 3-4 或 4-5）。
    • 由于子串不能重叠，例如选择了索引 3-4 的 00，则索引 4 不能再参与其他配对。
    • 最优解为 2 个子串（例如 11 和 00）。

因此，最大个数为 2。

解题思路

1. 暴力枚举的局限性

最直观的思路是暴力搜索。假设字符串中有 $m$ 个 ?，每个 ? 有 2 种填法，则总共有 $2^m$ 种可能的字符串。对于每种字符串，再遍历统计最大子串数。
缺点：当 $m$ 较大时（例如 $m=20$），计算量将突破百万级，导致超时。暴力法在此题中不可行。

2. 贪心策略

我们需要寻找一种局部最优策略，使得全局结果达到最优。

核心思考：如果我们正在处理某个位置 $i$，它是否应该与前面的位置 $i-1$ 配对，还是留给后面的位置 $i+1$ 配对？

贪心原则：优先与前面的字符配对。

理由：

  • 如果位置 $i$ 选择和 $i-1$ 配对，那么位置 $i$ 被占用，无法再与 $i+1$ 配对。
  • 如果位置 $i$ 选择不和 $i-1$ 配对，而是等待与 $i+1$ 配对，那么位置 $i-1$ 可能就永远无法配对了（因为 $i$ 是它唯一的机会）。
  • 显然，“确保前面的字符有机会配对”比“赌后面的字符能配对”更稳妥。因为配对成功就能贡献 +1 的价值，尽早拿下已知的收益是贪心的精髓。

具体操作：

  1. 从左到右遍历字符串。
  2. 维护一个数组或标记，记录每个位置是否已被使用。
  3. 对于位置 $i$：
    • 如果 $i$ 或 $i-1$ 已被使用，则跳过。
    • 如果字符相同（或者是 ? 通配），则配对成功，标记 $i$ 和 $i-1$ 为已使用，答案 +1。
    • 如果不能与前配对，则暂时保留，等待后续处理（实际上在遍历过程中，不与前配对自然就留给了后面，但逻辑上我们只需每一步看前面即可）。

关于 ? 的处理：

  • ? 是万能字符。
  • 如果当前字符是 ?，只要前一个字符存在且未被占用，就可以强制填充为与前一个字符相同的值，完成配对。
  • 如果当前字符是确定值（0 或 1），则必须前一个字符与之相同才能配对。

代码实现

以下是 Python 代码实现，完全遵循上述贪心逻辑：

```python
s = input()
ans = 0

li数组用于标记索引i位置是否已经被匹配过

初始化为False，表示所有位置均未被匹配

li = [False for _ in range(len(s))]

for i in range(len(s)):

# 如果当前位置已经被匹配过，跳过
if li[i]:
    continue

# 尝试与前一个位置(i-1)匹配
if i - 1 >= 0 and not li[i - 1]:
    matched = False

    # 情况1: 前一个字符是 '?'
    # 此时'?'可以变成任意字符，必然能和当前字符匹配
    if s[i - 1] == '?':
        matched = True
    # 情况2: 当前字符是 '?'
    # 此时'?'可以变成前一个字符，必然能匹配
    elif s[i] == '?':
        matched = True
    # 情况3: 两个字符都是确定值且相等
    elif s[i] == s[i - 1]:
        matched = True

    if matched:
        li[i] = True
        li[i - 1] = True
        ans += 1
        # 匹配成功后，当前位置i已使用，无需再向后匹配
        continue

# 注意：根据贪心策略，我们优先匹配前面。
# 如果无法与前面匹配，该字符将保留，等待被后一个字符匹配。
# 在循环的下一步(i+1)时，会尝试与当前的i匹配。
# 因此，不需要显式写“向后匹配”的逻辑。

print(ans)

复杂度分析

  • 时间复杂度：$O(n)$。我们只需遍历一次字符串，每次处理仅涉及常数时间的判断和标记。
  • 空间复杂度：$O(n)$。需要一个长度为 $n$ 的布尔数组来标记状态。

总结

本题是一道典型的贪心算法应用题。难点在于克服暴力枚举的思维定势，转而寻找局部最优解。通过“优先匹配左侧”的策略，我们保证了每一个字符尽可能早地贡献价值，从而得到全局最优解。这也启示我们在解决字符串匹配或区间问题时，单向扫描配合贪心策略往往能有效降低时间复杂度。
