Python爬虫基础：urllib代理的基本使用

发表于 2024-07-26 更新于 2026-05-14 分类于 Python爬虫

前言

在网络爬虫开发中，频繁请求同一网站往往会触发反爬机制，导致IP被封禁。为了解决这一问题，使用代理IP是常见的反反爬策略之一。本文将详细介绍如何利用 Python 标准库 urllib 设置代理，实现隐藏真实 IP 的请求。

核心概念

在 urllib 中使用代理，主要涉及以下三个核心对象：

  1. ProxyHandler：代理处理器，用于设置代理IP信息。
  2. build_opener：构建自定义的 opener 对象，它会将代理处理器加载进去。
  3. opener.open()：使用构建好的 opener 发送请求，此时请求将通过代理服务器转发。

代码实现

下面是一个完整的示例，演示如何通过代理访问百度搜索“ip”，并将结果保存到本地。

```python
import urllib.request

1. 设置目标URL和请求头

url = ‘http://www.baidu.com/s?wd=ip‘
headers = {
‘User-Agent’: ‘Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36’
}

2. 创建 Request 对象

req = urllib.request.Request(url=url, headers=headers)

3. 设置代理IP

注意：此处使用的代理IP为示例，实际使用时请替换为有效的代理IP

格式：{‘协议类型’: ‘IP地址:端口号’}

proxies = {
‘http’: ‘125.77.25.177:8090’
}

4. 创建 ProxyHandler 处理器

handler = urllib.request.ProxyHandler(proxies=proxies)

5. 创建自定义 opener 对象

opener = urllib.request.build_opener(handler)

6. 发送请求

此时不再使用 urllib.request.urlopen(req)，而是使用 opener.open(req)

try:
res = opener.open(req, timeout=5)

# 7. 获取响应内容
content = res.read().decode('utf-8')

# 8. 持久化存储
with open('daili.html', 'w', encoding='utf-8') as f:
    f.write(content)

print("页面抓取成功，已保存为 daili.html")

except Exception as e:
print(f”请求失败，原因：{e}”)

步骤详解

1. 构建请求对象

首先，我们需要定义目标 URL 和 User-Agent。将 User-Agent 封装进 Request 对象，可以伪装成浏览器访问，避免被简单的反爬拦截。

2. 定义代理字典

proxies 字典定义了代理服务器的协议和地址。

  • 如果是 HTTP 代理，键名为 'http'。
  • 如果是 HTTPS 代理，键名为 'https'。
  • 值的格式必须为 IP:端口。

3. 创建 Handler 和 Opener

这是最关键的一步。urllib 默认的 urlopen 方法不支持直接传入代理。我们需要：

  1. 使用 ProxyHandler 传入代理字典，生成一个处理器 handler。
  2. 使用 build_opener(handler) 构建一个自定义的 opener。这个 opener 此时已经具备了使用指定代理的能力。

4. 发送请求与保存

使用 opener.open(req) 发送请求。请求成功后，读取响应数据并将其写入 daili.html 文件。打开该文件，如果代理设置正确且有效，你应该能看到百度搜索“ip”的结果页面，且显示的 IP 地址应为代理 IP，而非你的真实 IP。

注意事项

  1. 代理有效性：代码中示例的代理 IP 可能已失效，请务必替换为你自己的可用代理。建议购买付费代理或搭建自用代理池。
  2. 超时处理：使用代理请求通常比直连慢，且容易连接失败。建议在 opener.open() 中添加 timeout 参数，并使用 try-except 捕获异常。
  3. 协议匹配：确保代理协议（http/https）与目标 URL 协议一致，或者配置支持两种协议的代理。

总结

通过 ProxyHandler 和 build_opener，我们可以灵活地在 urllib 中实现代理请求。这是爬虫进阶的重要一步，理解 Handler 和 Opener 的关系，对于后续学习 Cookie 处理、异常处理等高级功能至关重要。

Python 爬虫基础：urllib 模块快速上手

发表于 2024-07-26 更新于 2026-05-14 分类于 Python爬虫

前言

在 Python 爬虫开发中，urllib 是 Python 内置的标准库，无需额外安装即可使用。它提供了丰富的接口用于处理 URL 和网络请求。本文将通过一个简单的示例——获取百度首页源码，带你快速掌握 urllib 的基本使用流程。

完整代码示例

下面是一段最基础的爬虫代码，实现了向百度发送请求并获取页面 HTML 源码的功能：

```python

1. 导入 urllib 库的 request 模块

import urllib.request

2. 定义目标 URL (即你要访问的地址)

url = ‘http://www.baidu.com‘

3. 模拟浏览器向服务器发送请求

response 对象包含服务器返回的所有信息

response = urllib.request.urlopen(url)

4. 获取响应中的页面源码

read() 方法返回的是字节形式的二进制数据

需要使用 decode(‘utf-8’) 将二进制数据转换为字符串

content = response.read().decode(‘utf-8’)

5. 打印获取到的内容

print(content)

代码核心逻辑解析

1. 发送请求

``python response = urllib.request.urlopen(url)urlopen是urllib.request中最核心的方法，它可以像浏览器一样向指定的 URL 发送 GET 请求。返回的response` 对象包含了服务器的响应状态、响应头以及响应体。

2. 读取与解码

```python
content = response.read().decode(‘utf-8’)
这一步包含两个关键操作：

  • read(): 读取响应体的内容。值得注意的是，该方法返回的是二进制数据，通常格式为 <class 'bytes'>。直接打印会显示为 b'...' 的形式，可读性差。
  • decode('utf-8'): 为了将二进制数据还原成人类可读的字符串，我们需要进行解码。因为百度首页的编码格式通常为 UTF-8，所以这里使用 UTF-8 进行解码。

知识点总结

步骤 方法/变量 说明
导入模块 import urllib.request 引入处理请求的标准库。
定义地址 url 目标网站的统一资源定位符。
发起请求 urllib.request.urlopen(url) 向服务器发起 HTTP 请求。
读取数据 response.read() 获取二进制格式的网页源码。
解码数据 .decode('utf-8') 将二进制字节流转换为字符串。

结语

通过以上步骤，我们成功实现了获取网页源码的第一步。虽然 urllib 功能强大，但在实际开发中，由于其 API 相对底层，许多开发者也会选择使用更加便捷的第三方库如 requests。但理解 urllib 的原理对于掌握 Python 爬虫至关重要。

Python爬虫实战：使用urllib与XPath解析百度搜索按钮

发表于 2024-07-26 更新于 2026-05-14 分类于 技术实战

前言

在 Python 爬虫开发的入门阶段，学会如何发送请求并解析数据是核心技能。本文将介绍如何利用 Python 内置库 urllib 发送带伪装头的请求，并结合 lxml 库使用 XPath 语法，精准提取百度首页“百度一下”按钮的文本内容。

核心思路

  1. 发送请求：构建带有 User-Agent 的请求头，模拟浏览器访问，避免被服务器拒绝。
  2. 获取数据：使用 urlopen 获取响应，并解码为 UTF-8 格式的字符串。
  3. 解析数据：利用 lxml.etree 将 HTML 文本转换为可解析的对象，通过 XPath 路径表达式定位目标元素。

完整代码实现

以下是本次实战的完整代码：

```python

导入所需模块

import urllib.request
from lxml import etree

1. 定义目标URL

url = ‘https://www.baidu.com‘

2. 构建请求头 (伪装浏览器)

这里的 User-Agent 模拟了 Edge 浏览器

headers = {
‘User-Agent’: ‘Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36 Edge/16.16299’
}

3. 创建请求对象并添加头部信息

request = urllib.request.Request(url=url, headers=headers)

4. 发送请求并获取响应

response = urllib.request.urlopen(request)

5. 读取响应内容并解码

百度网页通常使用 UTF-8 编码

content = response.read().decode(‘utf-8’)

6. 解析网页源码

html = etree.HTML(content)

7. 使用 XPath 提取数据

逻辑：查找所有 id=”su” 的 input 标签，并获取其 value 属性值

xpath 返回的是一个列表，通过 [0] 取出第一个元素

result = html.xpath(‘//input[@id=”su”]/@value’)[0]

8. 打印结果

print(result)

代码详解

1. 为什么要设置 Headers？

直接使用 urllib 访问百度时，Python 会发送默认的 User-Agent（通常包含 Python-urllib 字样）。百度服务器识别到这是爬虫请求，可能会拒绝服务或返回不完整的数据。因此，我们需要在请求头中添加 User-Agent，伪装成正常的浏览器访问。

2. XPath 解析逻辑

代码中的核心解析语句是：
```python
result = html.xpath(‘//input[@id=”su”]/@value’)[0]

  • //input：在整个文档中查找 input 标签。
  • [@id="su"]：筛选条件，找到 id 属性为 "su" 的节点。在百度首页，这正好对应搜索框右侧的“百度一下”按钮。
  • /@value：获取该节点的 value 属性值。

运行程序后，控制台将输出：
百度一下

总结

本案例展示了最基础的爬虫流程。虽然 urllib 是 Python 内置库，但在实际开发中，我们通常会使用更加人性化的第三方库（如 requests）来处理请求。不过，理解 urllib 的底层逻辑对于掌握 Python 网络编程大有裨益。

  4. 代码可读性提升：

Python爬虫进阶：urllib请求对象定制与UA伪装

发表于 2024-07-26 分类于 Python爬虫

前言

在使用 Python 进行网络爬虫开发时，urllib 库是我们最常接触的基础库。很多初学者在使用 urllib.request.urlopen 时会发现：直接请求往往无法获取网页数据，或者被服务器重定向到登录页/错误页。

这通常是因为我们忽略了 请求头 的设置。本文将深入讲解如何通过 urllib 定制请求对象，实现 UA 伪装，以及如何处理 URL 中的中文编码问题。

一、基础知识：URL 的组成与端口

在深入代码之前，我们需要理解 URL 的基本构成。一个完整的 URL 包含协议、主机、端口号、路径、参数和锚点。

在爬虫请求中，端口号 往往决定了协议类型：

协议类型 默认端口 说明
HTTP 80 超文本传输协议
HTTPS 443 安全套接字层超文本传输协议
MySQL 3306 数据库服务端口
Redis 6379 缓存服务端口
MongoDB 27017 数据库服务端口

我们在代码中指定的 https://www.baidu.com，浏览器实际上默认连接的是 443 端口。

二、痛点：为什么需要请求对象定制？

urlopen 方法虽然简单，但它只接受 URL 字符串或特定的 Request 对象。

如果我们直接发起请求，urllib 发送的请求头中的 User-Agent 默认是 Python-urllib/x.x。绝大多数网站会识别出这是一个脚本请求，从而通过反爬机制直接拒绝访问。

核心问题：urlopen 方法本身不支持直接传递 headers 字典参数。

解决方案：使用 urllib.request.Request 类定制请求对象，将 URL 和 Headers 封装在一起。
