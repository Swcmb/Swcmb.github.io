[洛谷 P12281] 记事本 - 洛谷

题意简述

给定一个字符串，要求我们通过一系列操作（插入、删除和移动）来达到目标字符串。题目中提供了一些示例，展示了如何通过这些操作将原始字符串转换为目标字符串。例如，输入字符串为 “abcdefg”，目标字符串为 “abcd”，可以通过以下步骤实现：

  1. 插入 “c” 到索引 0 的位置；
  2. 删除索引 3 位置的字符；
  3. 向左移动索引 1 位置。

请尝试使用上述思路，编写代码实现这个功能。

思路分析

朴素解法

首先，我们可以使用一个简单的循环遍历字符串，然后根据当前索引进行相应的操作。这种方法的时间复杂度为 O(n)，空间复杂度为 O(1)。但是，这种方法在处理大量数据时可能会超时，因为每次操作都需要遍历整个字符串。

优化方向

为了提高算法的效率，我们可以采用以下策略：

  1. 使用前缀和数组存储每个字符出现的次数；
  2. 根据当前索引计算需要插入或删除的字符；
  3. 使用二分查找找到目标字符串的起始位置。

最终解法

下面是一个使用 Python 实现的解法：

```python
def solve(s, target):
prefix_sum = [0] * (len(s) + 1)
for i in range(1, len(s) + 1):
prefix_sum[i] = prefix_sum[i - 1] + s[i - 1]

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

index = binary_search(prefix_sum, target)
if index == -1:
    return False

result = []
insert_pos = index
delete_pos = index
for i in range(len(s)):
    if s[i] == 'c':
        result.insert(insert_pos, 'c')
        delete_pos += 1
    elif s[i] == 'd':
        result.append('d')
        delete_pos += 1
    elif s[i] == 'h':
        result.insert(insert_pos, 'h')
        delete_pos += 1
    elif s[i] == 'l':
        result.insert(insert_pos, 'l')
        delete_pos += 1
    insert_pos += 1
    delete_pos += 1

return ''.join(result) == target

复杂度分析

  • 时间复杂度：O(n)，其中 n 是字符串的长度。
  • 空间复杂度：O(n)，用于存储前缀和数组。

P12340 [蓝桥杯 2025 省 AB/Python B 第二场] 旗帜 题解

1. 题意简述

本题要求我们计算给定宽度 ( w ) 和高度 ( h ) 的旗帜上，字母 ‘A’ 出现的次数。旗帜的图案由字符串 s_li 中的字符按顺序排列组成，每个字符串的长度为 7，共有 8 个字符串。

2. 思路分析

  • 朴素解法：我们可以直接按照题目给出的字符串 s_li，逐个字符地拼接出旗帜，然后遍历旗帜上的每个字符，统计 ‘A’ 的出现次数。这种方法的时间复杂度为 ( O(hw) )，空间复杂度为 ( O(1) )，对于较大的 ( h ) 和 ( w ) 可能会超时。
  • 优化方向：由于字符串 s_li 的长度固定为 7，我们可以考虑使用循环来重复使用字符串，而不是每次都拼接新的字符串。

  • 最终解法：我们可以使用循环来重复使用字符串 s_li，每次循环拼接 ( w // 7 ) 次字符串，然后再加上字符串 s_li 的前 ( w \% 7 ) 个字符。这样，我们就可以避免每次都拼接新的字符串，从而减少时间复杂度。具体的转移方程如下：
    [
    \text{temp} = \text{s_li}[i \% 7] \times (w // 7) + \text{s_li}[i \% 7][: (w \% 7)]
    ]
    其中 ( i ) 是当前循环的索引。

    在遍历 temp 字符串时，我们可以直接统计 ‘A’ 的出现次数。

3. 复杂度分析

  • 时间复杂度：( O(hw) )，因为我们需要遍历整个旗帜。
  • 空间复杂度：( O(1) )，因为我们只需要存储一个长度为 7 的字符串和用于统计 ‘A’ 的计数器。

4. 核心代码 + 注释

```python
h, w = map(int, input().split())
s_li = [‘LANQIAO’, ‘ANQIAOL’, ‘NQIAOLA’, ‘QIAOLAN’, ‘IAOLANQ’, ‘AOLANQI’, ‘OLANQIA’, ‘LANQIAO’]
res = 0
for i in range(h):
s = s_li[i % 7]
temp = s * (w // 7) + s[:(w % 7)]
res += temp.count(“A”)
print(res)

  • 变量名：h 和 w 分别表示旗帜的高度和宽度，s_li 是包含旗帜图案的字符串列表，res 用于存储 ‘A’ 的出现次数。
  • 循环：循环遍历旗帜的每一行。
  • 字符串拼接：根据当前行的索引，选择对应的字符串，并拼接出当前行的图案。
  • 统计 ‘A’ 的出现次数：使用 count 方法统计当前行中 ‘A’ 的出现次数，并将其累加到 res 中。

5. 输入输出示例

输入：
3 7
输出：
4
解释：旗帜的图案为 LANQIAOANQIAO，其中 ‘A’ 出现了 4 次。

6. 易错点 & 收获

  • 易错点：在拼接字符串时，需要考虑余数的情况，确保不会出现越界。
  • 收获：通过本题，我们学习了如何使用循环和字符串操作来优化算法，并且加深了对字符串拼接和统计的理解。

题意简述

给定两个整数 ( n ) 和 ( m )，我们需要对从 1 到 ( n ) 的所有整数进行排序，使得它们的数位之和最小。输出排序后的第 ( m ) 小的整数。

思路分析

朴素解法

最直观的方法是计算每个数的数位之和，然后根据这个和进行排序。具体步骤如下：

  1. 计算从 1 到 ( n ) 的每个整数的数位之和。
  2. 将这些数按数位之和从小到大排序。
  3. 输出排序后的第 ( m ) 小的整数。

这种方法的不足在于：

  • 时间复杂度：计算每个数的数位之和需要 ( O(n \times k) ) 时间，其中 ( k ) 是整数的最大位数。
  • 空间复杂度：需要额外的空间存储每个数的数位之和和排序结果。

优化方向

我们可以通过预处理每个数的数位之和来优化算法。具体来说，我们可以使用前缀和数组来快速计算任意区间的数位之和。

最终解法

我们可以定义一个辅助函数 a(x) 来计算一个数的数位之和。然后，我们使用前缀和数组 prefix_sum 来快速计算任意区间的数位之和。具体步骤如下：

  1. 计算每个数的数位之和并存储在数组 sums 中。
  2. 使用前缀和数组 prefix_sum 来快速计算任意区间的数位之和。
  3. 对 sums 进行排序，并输出排序后的第 ( m ) 小的整数。

```python

m = int(input())

def a(x):
x = str(x)
res = 0
for i in x:
res += int(i)
return res

sums = []
for i in range(1, n + 1):
sums.append((i, a(i)))

计算前缀和数组

prefix_sum = [0] * (n + 1)
for i in range(1, n + 1):

对数位之和进行排序

sums.sort(key=lambda x: x[1])

复杂度分析

  • 时间复杂度：计算数位之和需要 ( O(n \times k) ) 时间，其中 ( k ) 是整数的最大位数。排序需要 ( O(n \log n) ) 时间。总的时间复杂度为 ( O(n \times k + n \log n) )。
  • 空间复杂度：需要额外的空间存储数位之和和前缀和数组，空间复杂度为 ( O(n) )。

def fourSum(nums, target):
prefixSum = [0] * (len(nums) + 1)
for i in range(len(nums)):
prefixSum[i+1] = prefixSum[i] + nums[i]
result = []
for i in range(len(nums) - 3):
left, right = i + 1, len(nums) - 1
while left < right:
sum = prefixSum[i] + prefixSum[left] + prefixSum[right] + prefixSum[i + 3]
if sum == target:
result.append((nums[i], nums[left], nums[right], nums[i + 3]))
while left < right and nums[left] == nums[left + 1]:
left += 1
while left < right and nums[right] == nums[right - 1]:
right -= 1
left += 1
right -= 1
elif sum < target:
left += 1
else:
right -= 1
return result

洛谷 P12843 生日相遇问题详解

2. 题意简述

给定一个日期范围，计算在这个范围内，某两个特定日期落在同一星期的年份数量。

输入内容：月份 m, 日 d1, 月份 n, 日 d2, 年份范围 k（从2025年开始计算）。

3. 思路分析

  • 朴素解法：遍历指定年份范围内的每一年，对于每一年的两个日期，使用基姆拉尔森计算公式计算星期几，然后比较两个日期是否落在同一星期。这种方法的时间复杂度为 $O(n \times k)$，其中 n 为年份范围内的年数，k 为年份范围内的年数，显然效率低下。
  • 优化方向：由于年份范围固定，可以考虑使用数学方法来减少比较次数。例如，我们可以计算两个日期相对于2025年的星期数，然后比较这两个星期数是否相同。
  • 最终解法：使用基姆拉尔森计算公式计算两个日期相对于2025年的星期数，然后比较这两个星期数是否相同。如果相同，则记录下这一年。这种方法的时间复杂度为 $O(k)$，空间复杂度为 $O(1)$。

4. 复杂度分析

  • 时间复杂度：$O(k)$，其中 k 为年份范围内的年数。

[洛谷 P13893] 工作时长 - 洛谷

题意简述

在洛谷平台上，题目P13893要求我们计算某人的工作时间。具体来说，我们需要读取一系列的时间点，每个时间点代表一个工作时段的开始或结束，然后计算出总的工作时长。

思路分析

朴素解法

最直观的方法是遍历所有时间点，对于每一对相邻的时间点，计算它们之间的差值并累加。然而，这种方法的时间复杂度为$O(n^2)$，在数据量较大时会导致超时。

优化方向

我们可以通过排序和双指针技术来优化算法。首先，对时间点进行排序，然后使用两个指针分别指向每对相邻的时间点，计算它们之间的差值并累加。这样，我们只需要遍历一次时间点列表，时间复杂度降低到$O(n \log n)$。

最终解法

我们定义两个指针i和j，分别指向当前考虑的开始和结束时间点。对于每一对相邻的时间点dt[i]和dt[j]，我们计算它们之间的差值并累加到总时长中。然后，我们将j指针向前移动一位，继续考虑下一对时间点。当i和j相遇时，所有时间点都已考虑完毕。

```python
from datetime import datetime

with open(“1.in”,”r”) as f:
l = f.readlines()
dt = sorted([datetime.fromisoformat(i.strip() for i in l)])
secs = 0
for i in range(0, len(dt) - 1, 2):
secs += (dt[i + 1] - dt[i]).seconds
print(secs)

复杂度分析

  • 时间复杂度：$O(n \log n)$，主要由排序步骤决定。
  • 空间复杂度：$O(n)$，用于存储时间点列表。

核心代码 + 注释

```python

使用双指针遍历时间点对

输出总时长

输入输出示例

输入：
1 4
2 5
7 8
输出：
3
解释：
时间段为[2, 5]和[7, 8]，总时长为$5 - 2 + 8 - 7 = 3$秒。

  • 数组越界：确保在遍历时间点时，i和j的值不会超出列表的范围。
  • 特殊数据处理：注意时间格式的一致性，确保所有时间点都能正确解析为datetime对象。

洛谷 P15308 等差数列计数 - 蓝桥杯 2025国 Python B OJ

题目链接或来源

洛谷 P15308

题面简述

这是一个关于等差数列的问题。给定一个序列，要求计算从 1 到 2025 中任选三个不同的数，能组成多少个不同的、长度为 3 的等差数列。注意这里“不同”包含顺序，即 (1,2,3) 和 (3,2,1) 算作两个不同的等差数列。

思路分析

朴素解法

首先，我们考虑最简单的方法：枚举所有可能的递增等差数列。对于每个递增等差数列，我们反转顺序得到一个递减等差数列，因此总数是递增数列数量的两倍。

优化方向

然而，这种方法的时间复杂度和空间复杂度都较高。为了提高效率，我们可以采用以下策略：

  • 固定公差：只考虑公差为正的情况，因为如果公差为负，那么数列将无法形成等差数列。

题意简述

本题是关于计算从1到n之间所有特别数的和。特别数是指各位数字之和为完全平方数的数。例如，121是一个特别数，因为1+2+1=4，而4是一个完全平方数。

思路分析

朴素解法

最直观的方法是遍历从1到n的所有整数，检查每个整数的各位数字之和是否为完全平方数。如果是，则将其加到结果中。然而，这种方法的时间复杂度为$O(n \times k)$，其中$k$是数字的最大位数，对于较大的$n$，这种方法会非常慢。

优化方向

我们可以通过预先计算所有可能的完全平方数来优化这个过程。具体来说，我们可以生成所有小于等于9999（因为最大的三位数各位数字之和为9+9+9=27）的完全平方数，并存储在一个集合中。然后，在遍历从1到n的整数时，检查其各位数字之和是否在集合中。

最终解法

我们定义一个辅助函数a(num)，用于判断一个数是否为特别数。然后，我们遍历从1到n的所有整数，使用a(num)函数检查其是否为特别数，并累加到结果中。

```python
def a(num):
s = str(num)
return any(int(digit) ** 2 == sum(map(int, s)) for digit in s)

n = int(input())

  • 时间复杂度：预处理完全平方数的时间复杂度为$O(1)$，遍历从1到n的时间复杂度为$O(n)$，因此总的时间复杂度为$O(n)$。
