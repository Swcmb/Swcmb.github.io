编写C语言程序时必须有#include这一行吗？为什么有时候不需要？

发表于 2024-11-26 分类于 编程基础 ， C语言

本文旨在解决 C 语言初学者对 #include 的常见困惑。由于市面上部分教材编排逻辑的缺失，导致很多初学者陷入了“学编程要先学 C，学 C 要先懂编程”的死循环。本文将从模块化设计的角度，重新审视 C 语言程序的结构。

前言：打破教材编排的“Bug”

市面上的 C 语言教程，往往存在一个严重的逻辑缺陷：无脑陈列细节，缺乏主线逻辑。

这种风气甚至可以追溯到 C 语言之父 Dennis Ritchie 的经典著作《C程序设计语言》。严格来说，这本书是一本“C 语言特性手册”，面向的是“初学 C 语言的程序员”，而非“编程零基础初学者”。然而，后世的许多教材（包括《C Primer Plus》等）都沿用了这种“字典式”的编排顺序，导致初学者在 Hello World 阶段就迷失了方向。

本文尝试打破这一惯例，通过第一课就讲清楚 #include，引出模块化设计的核心思想，帮助你建立从一行代码到完整项目的宏观认知。

第一课：#include 与模块化编程

程序即机器

一个优秀的程序设计，就像设计一台复杂的机器。它不是一块铁板，而是由多个功能模块组合而成。这些模块各司其职，最终由一个主体程序（通常是 main）统一调度。

让我们以编写一个“C 语言玩具车”程序为例，看看模块化思维是如何运作的。

1.0 版：单文件原型

在最原始的阶段，我们将所有功能模块写在一个文件中。这虽然简单，但已经体现了“模块化”的雏形——main 函数作为指挥官，负责调度各个功能函数。

  1. 如果你自己实现了所需功能：
    printf 并不是 C 语言语法的核心，它只是一个库函数。如果我自己写了一个打印函数，或者使用其他库，就不需要 stdio.h。

总结

  • #include <stdio.h> 不是仪式：它不是写 C 程序的“起手式”或魔法咒语。
  • 按需引入：你需要用什么功能，就包含对应的头文件。不用键盘操作，就不需要 <keyboard.h>（假设存在）；不用网络操作，就不需要 <network.h>。
  • 设计思维：学习 C 语言，不应从死记硬背 #include 开始，而应从理解“我要造一台什么样的机器”以及“这台机器由哪些模块组成”开始。

当你理解了模块化设计，#include 就不再是冰冷的代码行，而是你组装机器时拧下的一颗螺丝钉，用在哪里，就在哪里拧紧。

数据结构学习笔记：图论之七桥问题与欧拉回路（C++实现）

发表于 2024-10-30 更新于 2026-05-14 分类于 计算机科学 ， 数据结构

一、背景引入：哥尼斯堡七桥问题

七桥问题是图论发展史上的经典案例，由著名数学家欧拉在18世纪提出。问题的背景源于普鲁士的哥尼斯堡（现俄罗斯加里宁格勒），普雷格尔河穿过该城，形成了四个区域（两个岛屿及两岸），这四个区域由七座桥连接。

问题核心：是否可以从某个区域出发，经过所有七座桥，且每座桥只经过一次，最后回到起点？

问题的图论模型

为了解决这一问题，我们可以将实际问题抽象为数学模型：

  1. 顶点：将四个区域抽象为图中的四个顶点。
  2. 边：将七座桥抽象为连接顶点的边。

根据图论中的判定规则，欧拉回路存在的条件如下：

  • 欧拉回路：通过图中每一条边且仅通过一次，最后回到起点的回路。存在的充要条件是图中所有顶点的度数均为偶数。
  • 欧拉路径：通过图中每一条边且仅通过一次，但不要求回到起点的路径。存在的条件是图中恰好有两个顶点的度数为奇数（起点和终点）。
  • 如果奇度数顶点的数量超过两个，则既不存在欧拉回路，也不存在欧拉路径。

七桥问题的求解

在七桥问题中，四个区域（顶点）连接桥梁的情况如下：

  • 两个区域各自连接了3座桥（度数为奇数）。
  • 另外两个区域各自连接了3座桥（度数为奇数）。（注：历史真实数据为四个区域度数均为奇数，或根据具体抽象方式不同而变化，但奇度数顶点数量必然超过2个）。

结论：由于奇度数顶点的数量超过两个（实际上四个顶点的度数均为奇数），因此不存在欧拉回路，也不存在欧拉路径。

二、C++ 代码实现

下面通过 C++ 程序来实现上述判定逻辑。我们定义一个 EulerCircuit 类，通过邻接矩阵存储图结构，并计算奇度数顶点的数量来进行判定。

代码实现

```cpp

include

using namespace std;

// 定义图的最大顶点数
const int MaxSize = 4;

class EulerCircuit {
public:
// 构造函数，初始化图
EulerCircuit(int a[][MaxSize], int n);

// 计算奇度数顶点的数量
int oddVertexNum();

private:
int mat[MaxSize][MaxSize]; // 邻接矩阵
int vertexNum; // 顶点数量
};

// 构造函数实现
EulerCircuit::EulerCircuit(int a[][MaxSize], int n) {
for (int i = 0; i < n; i++) {
for (int j = 0; j < n; j++) {
mat[i][j] = a[i][j];
}
}
vertexNum = n;
}

// 计算奇度数顶点数量的函数实现
int EulerCircuit::oddVertexNum() {
int count = 0;
int degree;

for (int i = 0; i < vertexNum; i++) {
    degree = 0;
    // 计算第i个顶点的度数（邻接矩阵第i行元素之和）
    for (int j = 0; j < vertexNum; j++) {
        degree = degree + mat[i][j];
    }

    // 判断度数是否为奇数
    if (degree % 2 != 0) {
        count++;
    }
}
return count;

}

int main() {
// 定义邻接矩阵表示的图
// 这里的矩阵对应文中描述的连接情况或具体的测试用例
int a[4][MaxSize] = {
{0, 1, 2, 2}, // 顶点0：连接顶点1(1座), 顶点2(2座), 顶点3(2座) -> 度数5 (奇数)
{1, 0, 1, 1}, // 顶点1：连接顶点0(1座), 顶点2(1座), 顶点3(1座) -> 度数3 (奇数)
{2, 1, 0, 0}, // 顶点2：连接顶点0(2座), 顶点1(1座) -> 度数3 (奇数)
{2, 1, 0, 0} // 顶点3：连接顶点0(2座), 顶点1(1座) -> 度数3 (奇数)
};

// 实例化对象
EulerCircuit G{a, 4};

// 获取奇度数顶点数量
int num = G.oddVertexNum();

// 输出判定结果
if (num >= 2) {
    // 注意：若num==2，存在欧拉路径；若num>2，不存在任何路径。
    // 此处代码逻辑简化了判定，实际应用中可细分。
    cout << num << "个地方通奇数桥，不存在欧拉回路" << endl;
} else {
    cout << "存在欧拉回路" << endl;
}

return 0;

}

代码解析

  1. 数据结构：使用二维数组 mat[MaxSize][MaxSize] 作为邻接矩阵存储图。mat[i][j] 的值表示顶点 $i$ 和顶点 $j$ 之间的边数（桥的数量）。
  2. 度数计算：在 oddVertexNum 函数中，通过双层循环计算每个顶点的度数。对于无向图，顶点 $i$ 的度数即为邻接矩阵第 $i$ 行所有元素之和。
  3. 判定逻辑：主函数中定义了一个具体的邻接矩阵。计算结果显示有4个奇度数顶点，根据欧拉回路判定定理，程序正确输出了“不存在欧拉回路”的结论。

三、总结

七桥问题不仅是图论的起源，也为我们提供了解决“一笔画”问题的数学模型。通过将实际问题抽象化，并利用编程手段计算顶点度数，我们可以轻松判断一个图是否存在欧拉回路。这体现了数据结构与算法在解决实际问题中的强大威力。

埃拉托斯特尼筛法：原理与代码实现

发表于 2024-09-23 更新于 2026-05-14 分类于 计算机科学

1. 算法简介

埃拉托斯特尼筛法（Sieve of Eratosthenes）是一种用于筛选素数的经典算法。该算法由古希腊数学家埃拉托斯特尼于公元前3世纪提出，至今仍是计算机科学入门数论算法的重要一环。其核心优势在于思路直观、实现简单，且在处理 $10^7$ 级别以下的数据时效率极高。

2. 核心原理

该算法基于一个简单的数学事实：一个质数的倍数必然不是质数。

如果我们从最小的质数开始，将其所有的倍数标记为合数，那么剩余未被标记的数即为质数。具体步骤如下：

  1. 初始化：创建一个从 $2$ 到 $n$ 的连续整数列表，初始时假设所有数均为质数（状态设为 True）。
  2. 筛选过程：
    • 找到当前最小的未被标记为合数的数 $p$，该数即为质数。
    • 将 $p$ 的所有倍数（$2p, 3p, \dots$）标记为合数（状态设为 False）。
  3. 优化边界：实际上，我们无需对 $p$ 筛选到最后。根据数学原理，只需筛选到 $p \le \sqrt{n}$ 即可。因为如果 $p > \sqrt{n}$，那么 $p \times p$ 已经大于 $n$，此时的倍数早已被更小的质数筛选过了。
  4. 输出结果：最终列表中状态仍为 True 的数即为所求质数。
```
