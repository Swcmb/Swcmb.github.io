蓝桥杯真题：最小砝码（贪心与数学推导)

发表于 2024-04-08 更新于 2026-05-14 分类于 Algorithm

问题描述

你有一架天平。现在你要设计一套砝码，使得利用这些砝码可以称出任意小于等于 $N$ 的正整数重量。那么这套砝码最少需要包含多少个砝码？注意砝码可以放在天平两边。

输入格式

输入包含一个正整数 $N$。

输出格式

输出一个整数代表答案。

题目分析与推导

这是一道经典的数学推导与贪心算法结合的题目。我们需要找到能覆盖 $[1, N]$ 范围内所有整数重量的最少砝码组合。

1. 贪心策略的选择

砝码可以放在天平的两端，这意味着对于每一个砝码，它有三种状态：

  1. 放在左盘（记为 $+w$）。
  2. 放在右盘（记为 $-w$）。
  3. 不放（记为 $0$）。

这实际上对应了平衡三进制（Balanced Ternary）的概念。

2. 为什么选择 $1, 3, 9, \dots$？

假设我们已经有一组砝码，能称出的最大重量为 $M$。当我们加入一个新的砝码 $w$ 时，由于砝码可以放在两边，我们能称出的新范围会发生变化。

设当前砝码能称出的最大重量为 $K$（即能覆盖 $[1, K]$ 的所有整数）。
当我们加入一个新砝码 $w$ 时，新的可称量范围变为：

  • 原有范围：$[1, K]$
  • 新增范围（利用新砝码 $w$）：
    • $w$ 单独：$w$
    • $w + [1, K]$：$[w+1, w+K]$
    • $w - [1, K]$（放在异侧）：$[w-K, w-1]$

3. 推导过程

根据上述推导，我们来确定砝码序列：

  1. 第一个砝码：初始 $K=0$，则 $w_1 = 2(0) + 1 = 1$。
    • 当前最大称重 $K_{new} = 1$。
  2. 第二个砝码：当前 $K=1$，则 $w_2 = 2(1) + 1 = 3$。
    • 当前最大称重 $K_{new} = 1 + 3 = 4$（覆盖 $1$ 到 $4$）。
  3. 第三个砝码：当前 $K=4$，则 $w_3 = 2(4) + 1 = 9$。
    • 当前最大称重 $K_{new} = 4 + 9 = 13$（覆盖 $1$ 到 $13$）。
  4. 第四个砝码：当前 $K=13$，则 $w_4 = 2(13) + 1 = 27$。
    • 当前最大称重 $K_{new} = 13 + 27 = 40$。

前 10 个幸运数字的十进制表示为：1, 2, 4, 6, 8, 40, 48, 72, 120, 126…

问题：请问第 2023 个幸运数字是多少？你只需要输出这个整数的十进制表示即可。

解题思路

本题是一道典型的枚举模拟题。由于我们需要找到第 2023 个符合条件的数，且该位置的数据规模在整型范围内，直接暴力枚举并判断即可。

具体步骤如下：

  1. 遍历整数：从 1 开始遍历所有正整数。
  2. 进制转换：对于每个整数，分别计算其在二进制、八进制、十进制、十六进制下的字符串表示。
    • Python 中可直接使用内置函数 bin(), oct(), hex()，注意它们生成的字符串带有前缀（如 0b, 0o, 0x），切片时需要去掉。
  3. 数位求和：编写一个函数，计算某个进制字符串下的各位数字之和。
    • 注意十六进制中包含 a-f，对应十进制的 10-15，需特殊处理。
  4. 整除判断：判断当前整数是否能被这四种进制下的数位和整除。
  5. 计数与输出：如果同时满足四种进制的哈沙德数条件，则计数器加 1。当计数器达到 2023 时，输出该数字并结束程序。

题目描述

在实际的开发工作中，对字符串的处理是最常见的编程任务之一。本题要求编写程序，对用户输入的字符串进行特定规则的处理。

具体处理规则如下：

  1. 首字母大写：将每个单词的首字母转换为大写。
  2. 数字字母分隔：在数字与字母之间插入下划线字符 _，以增强可读性（需同时考虑“字母在前，数字在后”和“数字在前，字母在后”两种情况）。
  3. 空格规范化：将单词间的一个或多个空格调整为 1个 空格。

输入描述

用户输入的字符串仅包含 小写字母、空格 和 数字，不含其他字符。

  • 单词间由 1 个或多个空格分隔。
  • 输入字符串长度不超过 200 个字符。

输出描述

输出经过上述规则处理后的字符串。

解题思路

本题考察字符串的基本操作，包括分割、遍历、类型判断与拼接。我们可以将问题拆解为以下三个步骤：

  1. 空格规范化：利用 Python 的 split() 方法不带参数的特性，自动将字符串按空白字符（包括多个空格、制表符等）分割成单词列表，同时去除了多余的空格。
  2. 数字字母分隔：遍历每个单词的字符。检查当前字符与下一个字符的类型，如果一个是数字一个是字母，则插入下划线。这里需要注意遍历的边界，避免索引越界。
  3. 首字母大写：在完成上述处理后，利用字符串的 upper() 方法将单词首字母大写，最后用 join() 方法将列表重新拼接成字符串。

蓝桥杯练习：图像模糊处理

发表于 2024-04-08 更新于 2026-05-14 分类于 算法练习

题目描述

小蓝有一张黑白图像，由 $n \times m$ 个像素组成，其中从上到下共 $n$ 行，每行从左到右 $m$ 列。每个像素由一个 $0$ 到 $255$ 之间的灰度值表示。

现在，小蓝准备对图像进行模糊操作，操作的方法为：对于每个像素，将以它为中心 $3 \times 3$ 区域内的所有像素（可能是 $9$ 个像素或少于 $9$ 个像素）求和后除以这个范围内的像素个数（取下整），得到的值就是模糊后的结果。

注意：每个像素都要用原图中的灰度值计算求和。

输入描述

输入的第一行包含两个整数 $n, m$。

第 $2$ 行到第 $n+1$ 行每行包含 $m$ 个整数，表示每个像素的灰度值，相邻整数之间用一个空格分隔。

数据范围：$1 < n, m < 100$。

输出描述

输出 $n$ 行，每行 $m$ 个整数，相邻整数之间用空格分隔，表示模糊后的图像。

题目分析与解题思路

这是一道典型的矩阵模拟题目，主要考察对二维数组的操作以及边界条件的处理。

核心步骤

  1. 数据读取：读取图像的行数 $n$ 和列数 $m$，并将像素数据存入二维列表（矩阵）中。
  2. 创建副本：由于题目要求“每个像素都要用原图中的灰度值计算”，这意味着我们不能在原矩阵上直接修改，否则会影响后续像素的计算。因此，需要创建一个同样大小的结果矩阵。
  3. 遍历计算：
    • 遍历矩阵中的每一个像素点 $(i, j)$。
    • 对于每个点，检查其周围 $3 \times 3$ 的邻域。邻域的范围是行 $[i-1, i+1]$ 和列 $[j-1, j+1]$。
    • 边界处理：当 $(i, j)$ 位于图像边缘（第一行、最后一行、第一列、最后一列）时，$3 \times 3$ 邻域会超出矩阵边界。此时需要判断坐标是否合法（$0 \le k < n$ 且 $0 \le l < m$）。
    • 求和与计数：累加邻域内合法像素的灰度值，并统计像素个数。
    • 取整赋值：计算平均值（向下取整）并赋值给结果矩阵对应位置。

时间复杂度

由于需要对每个像素遍历其周围的 $3 \times 3$ 邻域，且 $n, m$ 最大接近 $100$，总运算次数约为 $100 \times 100 \times 9 = 9 \times 10^4$，远小于一般限制的 $10^8$，算法效率完全满足要求。

无尽的石头

发表于 2024-04-08 更新于 2026-05-14 分类于 算法笔记

问题描述

在一个古老的迷宫中，有一道无尽的通道。通道上每隔一定的距离就会有一块神秘的石头，石头上刻着从 1 开始的连续整数。从 1 号石头开始，每块石头的编号都比前一块大 1。

石头上的数字拥有特殊的意义。如果你站在编号为 $n$ 的石头上，并向前走，你将会瞬间移动到编号为 $n+x$ 的石头上，其中 $x$ 为 $n$ 的各位数字之和。

举个例子：
如果你站在编号为 16 的石头上，由于 $1+6=7$，所以下一步你会移动到编号为 $16+7=23$ 的石头上。

现在，会有多次询问，你需要对每个询问输出从 1 号石头出发到达指定编号石头的最少步数。如果无法到达，则输出 -1。

输入格式

第一行输入一个整数 $t$ ($1 < t < 100$)，表示有 $t$ 次询问。

接下来 $t$ 行，每行一个整数 $n$ ($1 \le n \le 10^5$)，表示目标石头的编号。

输出格式

对于每个询问，输出一行，表示从 1 号石头到达目标石头的最少步数。如果无法到达，输出 -1。

思路分析

本题属于模拟与图论建模的结合问题。

  1. 模型抽象：我们可以将石头编号看作图中的节点。从节点 $n$ 出发，下一步只能到达节点 $n + \text{digits_sum}(n)$。这实际上构成了一棵以 1 为根节点的树（或者说是单链分支的集合）。
  2. 可达性分析：由于路径是单向且确定的，对于任意节点 $n$，它的前驱节点是唯一的（如果存在），后继节点也是唯一的。从 1 号石头出发的路径是固定的。
  3. 预处理与查询优化：
    由于询问次数 $t$ 可能较多，且目标编号 $n$ 具有随机性，我们可以采用预处理的方式：
    • 从 1 开始模拟移动过程，记录沿途经过的每一个节点（石头编号）以及到达该节点的步数。
    • 使用一个列表 reachable_stones 记录路径上的所有节点。
    • 对于用户的每一次询问 $n$：
      • 如果我们当前的模拟终点已经 $\ge n$，直接判断 $n$ 是否在 reachable_stones 中。若在，输出下标（即步数）；若不在，说明永远无法到达（因为路径是严格递增的），输出 -1。
      • 如果模拟终点 $< n$，继续向后模拟计算，直到终点 $\ge n$ 为止，然后进行判断。

这种方法利用了路径的单调递增性，避免了重复计算。
