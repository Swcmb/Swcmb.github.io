算法笔记：缩位求和的迭代与递归实现

发表于 2024-03-27 更新于 2026-05-14 分类于 算法笔记

题目描述

在电子计算机普及以前，人们经常使用一种粗略的方法来验算四则运算是否正确，这种方法类似于“弃九法”。

例如： 验算 248 x 15 = 3720

  1. 对乘数和被乘数分别逐位求和，若结果为多位数则继续逐位求和，直到变为一位数。
    • 248 -> 2+4+8=14 -> 1+4=5
    • 15 -> 1+5=6
    • 乘积验算：5 x 6 = 30 -> 3+0=3
  2. 对实际运算结果逐位求和：
    • 3720 -> 3+7+2+0=12 -> 1+2=3
  3. 若两边结果一致（均为3），说明计算正确的可能性很大。

任务： 请编写一个计算机程序，对给定的数字字符串进行反复逐位求和，直到结果为一位数。

输入输出描述

输入描述：
输入为一个数字字符串，表示一个 $n$ 位数（$n < 1000$）。

输出描述：
输出一位数，表示反复逐位求和的最终结果。

算法思路

本题的核心逻辑是将数字各位相加，如果结果大于等于10，则重复该过程。这是一个典型的重复计算问题，既可以使用迭代（循环）实现，也可以使用递归实现。

代码实现

1. 迭代写法

迭代算法通过循环结构重复执行操作，直到满足终止条件。在本题中，终止条件是数字仅剩一位。

核心逻辑：

  1. 定义一个函数 num_sum 计算当前数字的各位之和。
  2. 使用 while 循环判断当前数字位数是否大于1，若大于1则调用函数更新数值。

```python
def num_sum(a):
“””计算数字 a 的各位之和”””
a = str(a)
res = 0
for i in a:
res += int(i)
return res

读取输入

num = input().strip() # 建议直接读取字符串，避免大整数转换开销，逻辑处理时保持字符串或转int视情况而定
x = int(num) # 原逻辑转int处理

迭代求解

while len(str(x)) != 1:
x = num_sum(x)

print(x)

注： 原代码中使用 int(input()) 接收输入。由于题目保证 $n<1000$，Python 支持大整数，此处转换无碍，但在处理超大数字时，直接作为字符串操作往往更优。

2. 递归写法

递归算法将问题分解为更小的子问题。对于本题，如果数字已经是一位，直接返回；否则计算各位和并调用自身。

核心逻辑：

  1. 基准条件： 如果数字长度为1，直接返回该数字。
  2. 递归步骤： 计算各位之和，将和作为参数调用自身。

```python
def num_sum(a):
“””递归计算缩位求和”””

# 将输入转为字符串以便判断长度和遍历
s = str(a)

# 基准情况：如果只剩一位，返回结果
if len(s) == 1:
    return int(s)

# 递归步骤：计算各位和并递归调用
res = 0
for i in s:
    res += int(i)
return num_sum(res)

读取输入并调用

num = input().strip()
print(num_sum(num))

方法对比

特性 迭代法 递归法
实现方式 使用循环结构 函数调用自身
代码可读性 逻辑直观，结构清晰 代码简洁，更贴近数学定义
内存消耗 通常较低，无栈帧开销 每次调用需分配栈空间，深度过大可能栈溢出
适用场景 适用于大多数线性重复任务 适用于问题可自然分解为子问题的场景

对于本题，由于输入位数 $n < 1000$，递归深度非常有限（因为数字和衰减极快，如1000位的9，第一次求和也仅为9000），因此两种方法均非常安全且高效。

小蓝可以控制一个按钮同时打开所有的出水管，打开时，有出水管的位置可以被认为已经灌溉好。每经过一分钟，水就会向四面扩展一个方格，被扩展到的方格可以被认为已经灌溉好。即如果前一分钟某一个方格被灌溉好，则下一分钟它上下左右的四个方格也被灌溉好。

数据范围：
$1 \le n, m \le 100$
$1 \le t \le 10$

输出描述

输出一个整数，表示 $k$ 分钟后被灌溉好的方格数量。

蓝桥杯练习：天干地支计算

发表于 2024-03-25 更新于 2026-05-14 分类于 算法笔记

题目背景

在中国古代，时间记录采用“干支纪年法”，即通过天干和地支的组合来表示年份。天干有十个，地支有十二个，它们按固定顺序搭配，循环记录年份。本题要求根据给定的公元纪年，计算出对应的天干地支。

思路解析

解决这道题的关键在于找出公元纪年与天干地支序列之间的数学映射关系。

1. 数据定义

首先，我们需要将天干和地支映射为列表（数组），索引从 0 开始：

  • 天干：甲、乙、丙、丁、戊、己、庚、辛、壬、癸。
  • 地支：子、丑、寅、卯、辰、巳、午、未、申、酉、戌、亥。

2. 寻找基准点

通过观察或查阅资料可知，公元 4 年恰好是“甲子年”。

  • 甲：天干列表索引 0
  • 子：地支列表索引 0

因此，我们可以将公元 4 年作为计算的基准点。

3. 算法推导

对于任意输入年份 year：

  1. 计算偏移量：offset = year - 4。这表示该年份距离基准年过去了多少年。
  2. 计算天干索引：由于天干有 10 个，循环周期为 10。
    • 索引 = offset % 10
  3. 计算地支索引：由于地支有 12 个，循环周期为 12。
    • 索引 = offset % 12

最后，根据计算出的索引在列表中取值并拼接即可。

蓝桥杯试题：金币问题题解

发表于 2024-03-25 更新于 2026-05-14 分类于 算法笔记 ， 蓝桥杯

题目描述

国王将金币作为工资，发放给忠诚的骑士。发放规则如下：

  • 第一天，骑士收到一枚金币；
  • 之后两天（第二天和第三天），每天收到两枚金币；
  • 之后三天（第四、五、六天），每天收到三枚金币；
  • 之后四天（第七、八、九、十天），每天收到四枚金币；
  • ……

这种工资发放模式会一直延续下去：当连续 $N$ 天每天收到 $N$ 枚金币后，骑士会在之后的连续 $N+1$ 天里，每天收到 $N+1$ 枚金币。

请计算在前 $K$ 天里，骑士一共获得了多少金币。

输入描述

输入只有 1 行，包含一个正整数 $K$ ($1 \le K \le 10^4$)，表示发放金币的天数。

输出描述

输出只有 1 行，包含一个正整数，即骑士在前 $K$ 天里收到的金币总数。

题目分析

本题考察的是模拟或数学规律推导。

观察题目规律，我们可以发现金币发放是分阶段的：

  • 第 1 阶段：持续 1 天，每天 1 枚；
  • 第 2 阶段：持续 2 天，每天 2 枚；
  • 第 3 阶段：持续 3 天，每天 3 枚；
  • …
  • 第 $i$ 阶段：持续 $i$ 天，每天 $i$ 枚。

我们需要求的是前 $K$ 天的总金币数。我们可以模拟这个发放过程，构造一个长度足够涵盖 $K$ 天的列表，最后对前 $K$ 项求和即可。

复杂度分析：
由于 $K$ 最大仅为 $10^4$，通过两层循环模拟出每一天的金币数量，时间复杂度约为 $O(\sqrt{K})$ 级别（因为 $1+2+…+n \approx K$），在题目给定的数据范围内完全可以通过。

蓝桥杯经典题：扫雷游戏模拟

发表于 2024-03-24 更新于 2026-05-14 分类于 算法刷题

题目描述

在一个 $n$ 行 $m$ 列的方格地图上，部分位置埋有地雷，其余位置为空。
本题要求：对于每一个空位置，输出其周围八个相邻方格中地雷的数量；对于每一个地雷位置，直接输出数字 9。

输入输出格式

输入描述

  • 第一行包含两个整数 $n, m$，表示方格图的行数和列数。
  • 接下来的 $n$ 行，每行包含 $m$ 个整数（$0$ 或 $1$），整数之间用一个空格分隔。
    • 0 表示该方格没有地雷。
    • 1 表示该方格有地雷。

输出描述

输出 $n$ 行，每行 $m$ 个整数，相邻整数之间用空格分隔。

  • 对于没有地雷的方格，输出周围的地雷数量。
  • 对于有地雷的方格，输出 9。

解题思路

本题是一个典型的矩阵模拟问题。核心难点在于如何优雅地处理矩阵的边界检查。

1. 坐标偏移量

对于一个坐标 $(i, j)$，其周围的八个邻居坐标可以通过偏移量来表示：

方向 坐标偏移
左上 $(i-1, j-1)$
正上 $(i-1, j)$
右上 $(i-1, j+1)$
左侧 $(i, j-1)$
右侧 $(i, j+1)$
左下 $(i+1, j-1)$
正下 $(i+1, j)$
右下 $(i+1, j+1)$

2. 边界处理技巧

在遍历矩阵边缘的格子时，访问邻居容易发生数组越界。常见的解决方案有两种：

  1. 条件判断法：在访问邻居前，增加 if 语句判断坐标是否合法。
  2. 矩阵扩容法（推荐）：在输入矩阵的四周填充一层“虚拟边界”（例如全为0或无雷标记）。这样，原矩阵的所有真实格子的邻居都变成了有效下标，无需额外判断越界，代码更加简洁。

本文提供的代码采用了矩阵扩容法，在原始地图外围添加了一圈非雷标记，简化了逻辑。

代码实现

```python

读取行数 n 和列数 m

n, m = map(int, input().split())

初始化结果矩阵，全为 0

res = [[0 for in range(m)] for in range(n)]

构建地图，外围添加一圈“墙”（值为3），防止越界

这里填充 3 是因为 0 和 1 已被占用，只要不是 1 即可

maps = [[3 for _ in range(m + 2)]]

for _ in range(n):

# 每行读取数据，并在首尾添加边界值 3
row = [3] + list(map(int, input().split())) + [3]
maps.append(row)

添加底部边界

maps.append([3 for _ in range(m + 2)])

遍历实际地图区域（下标从 1 开始，到 n 结束）

for i in range(1, n + 1):
for j in range(1, m + 1):

    # 如果当前位置是地雷
    if maps[i][j] == 1:
        res[i-1][j-1] = 9
    else:
        # 统计周围 8 个方向的地雷数量
        temp = 0
        # 上方三个邻居
        if maps[i-1][j-1] == 1: temp += 1
        if maps[i-1][j] == 1:   temp += 1
        if maps[i-1][j+1] == 1: temp += 1
        # 左右两个邻居
        if maps[i][j-1] == 1:   temp += 1
        if maps[i][j+1] == 1:   temp += 1
        # 下方三个邻居
        if maps[i+1][j-1] == 1: temp += 1
        if maps[i+1][j] == 1:   temp += 1
        if maps[i+1][j+1] == 1: temp += 1

        res[i-1][j-1] = temp

按行输出结果

for row in res:
print(“ “.join(map(str, row)))

算法分析

  • 时间复杂度：$O(n \times m)$。我们需要遍历整个矩阵，每个格子的处理（统计周围8个邻居）是常数时间 $O(1)$，总体复杂度与矩阵大小呈线性关系。
  • 空间复杂度：$O(n \times m)$。需要额外的空间存储地图和结果矩阵。

蓝桥杯真题：平均 (贪心策略)

发表于 2024-03-23 更新于 2026-05-14 分类于 Algorithm

题目描述

问题描述

给定一个长度为 $n$ 的数组（$n$ 是 $10$ 的倍数），数组中的每个元素 $a_i$ 都是区间 $[0, 9]$ 中的整数。小明发现数组里每种数字出现的次数不太平均。

现在小明可以更改若干个数的值。已知更改第 $i$ 个数的代价为 $b_i$。小明希望通过更改，使得这 $10$ 种数字（$0 \sim 9$）出现的次数相等（即每种数字都出现 $n/10$ 次）。

请你计算并输出满足条件的最小代价和。

输入格式

输入的第一行包含一个正整数 $n$。
接下来 $n$ 行，第 $i$ 行包含两个整数 $a_i, b_i$，分别表示第 $i$ 个数的值和更改它的代价，用一个空格分隔。

输出格式

输出一行包含一个正整数，表示最小代价和。

样例输入

```text
10
1 1
1 2
1 3
2 1
2 2
2 3
3 1
3 2
3 3
4 1

样例输出

```text
3

题目分析

本题是一道典型的贪心算法应用题。

核心目标：我们需要让数字 $0 \sim 9$ 每个数字出现的次数都刚好等于 $k = n / 10$ 次。

解题思路：

  1. 数据分组：首先，我们需要统计每个数字（$0 \sim 9$）当前出现的次数以及它们对应的修改代价。我们可以创建一个列表 numbers，其中 numbers[i] 存储了所有数值为 $i$ 的修改代价列表。
  2. 贪心策略：
    • 对于某个数字 $i$，假设它当前出现的次数为 $count$，而目标次数为 $k = n/10$。
    • 如果 $count \le k$，说明该数字数量不足或刚好，不需要修改该数字，代价为 $0$。
    • 如果 $count > k$，说明该数字数量过多，我们需要将其中 $count - k$ 个数字修改为其他数字。
    • 关键点：为了使总代价最小，我们应该保留哪些数字不变？显然，我们应该保留修改代价最大的那些数字不变（因为保留它们意味着避免了最大的花费），而选择修改代价最小的那些数字将其变为其他值。
  3. 算法流程：
    • 遍历 $0 \sim 9$ 每个数字分组。
    • 对每个分组内的代价列表进行升序排序。
    • 如果该分组的大小 len(group) 大于 $k$，则计算前 len(group) - k 个代价之和（即最小的那些代价），累加到总答案中。
    • 如果分组大小小于等于 $k$，则不进行操作。

通过这种“局部最优”的选择，我们可以保证全局的总代价最小。

代码实现

```python
n = int(input())

创建一个包含10个列表的列表，用于存储数字0-9各自的修改代价

numbers[i] 将存放所有值为 i 的数字的修改代价 b

numbers = [[] for _ in range(10)]

读取输入数据并进行分组

for _ in range(n):
a, b = map(int, input().split())
numbers[a].append(b)

ans = 0 # 记录总的最小代价
k = n // 10 # 每种数字最终的目标数量

for i in range(10):

# 对当前数字 i 的所有修改代价进行升序排序
ls = sorted(numbers[i])

# 计算当前数字的数量
count = len(ls)

# 如果当前数字的数量超过了目标数量 k
# 我们需要修改掉 (count - k) 个数字
# 为了代价最小，我们选择修改代价最小的那些数字
# ls[:-k] 切片操作表示取列表的前 count - k 个元素（即最小的那些代价）
if count > k:
    ans += sum(ls[:-k])

print(ans)

复杂度分析

  • 时间复杂度：$O(n \log n)$。主要时间消耗在于排序。虽然我们对每个分组分别排序，但所有元素的总和为 $n$，平均情况下每个分组大小为 $n/10$，排序总复杂度仍在 $O(n \log n)$ 级别。
  • 空间复杂度：$O(n)$。需要存储每个数字的代价列表。
