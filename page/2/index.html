[洛谷 P12138] 寻找质数 - 洛谷

核心优化技巧

埃拉托斯特尼筛法 + 欧拉筛

题目概览

题目来源: 洛谷

题目难度: ★★★★☆

标签分类: 算法标签、数据结构标签

核心考点: 埃拉托斯特尼筛法、欧拉筛、质数定理

题意解析

题目要求找到第 (n) 个质数。输入一个整数 (n)，输出第 (n) 个质数。

输入格式: 一个整数 (n)。

输出格式: 输出第 (n) 个质数。

样例分析: 当 (n = 10) 时，输出第 10 个质数，即 29。

算法思路

4.1 问题分析

我们需要找到第 (n) 个质数。质数是指除了 1 和它本身外没有其他因数的自然数。我们可以使用埃拉托斯特尼筛法和欧拉筛来高效地找到这些质数。

4.2 算法原理

朴素思路: 最直观的解法是逐一检查每个数是否为质数，直到找到第 (n) 个质数。这种方法的时间复杂度为 (O(n \sqrt{n}))，空间复杂度为 (O(n))。

优化路径: 使用埃拉托斯特尼筛法或欧拉筛可以显著提高效率。埃拉托斯特尼筛法的时间复杂度为 (O(n \log \log n))，空间复杂度为 (O(n))。欧拉筛的时间复杂度为 (O(n))，空间复杂度为 (O(n))。

算法选择: 欧拉筛更适合，因为它保证了每个合数只被它的最小质因子标记一次，减少了重复标记。

核心思想: 欧拉筛通过维护一个质数列表，利用已知的质数来筛选合数，从而高效地找到质数。

4.3 数学推导

根据质数定理，第 (n) 个质数 (p_n) 近似满足 (p_n \approx n \ln n)。我们可以使用这个公式来估算一个足够大的范围，然后用筛法找出范围内所有质数，直接提取第 (n) 个。

4.4 多种解法对比

解法 时间复杂度 空间复杂度 适用场景 代码难度
埃拉托斯特尼筛法 (O(n \log \log n)) (O(n)) 小规模数据 简单
欧拉筛 (O(n)) (O(n)) 中等规模 中等

5.2 逐行解析

  • if n == 1: return 2: 如果 (n) 为 1，直接返回 2，因为 2 是第一个质数。
  • from math import log: 导入数学函数库，用于计算对数。
  • limit = int(n * (log(n) + log(log(n)))) if n >= 6 else 15: 计算初始界限，对于 (n \ge 6)，使用公式 (n \ln n)，否则使用 15。
  • while True: 循环直到找到第 (n) 个质数。
  • primes = euler_sieve(limit): 使用欧拉筛筛选出所有不超过 (limit) 的质数。
  • if len(primes) >= n: return primes[n-1]: 如果质数个数足够，返回第 (n) 个质数。
  • limit = int(limit * 1.5) + 1: 如果质数个数不足，扩大界限并继续筛选。

  • 时间复杂度: (O(n \log \log n)) 对于埃拉托斯特尼筛法，(O(n)) 对于欧拉筛。

输入输出示例

样例输入: 10

样例输出: 29

执行过程:

  1. 计算初始界限 limit = 10 * (log(10) + log(log(10))) ≈ 24.3，取整为 24。
  2. 使用欧拉筛筛选出所有不超过 24 的质数：[2, 3, 5, 7, 11, 13, 17, 19, 23]。
  3. 质数个数足够，返回第 10 个质数 29。

总结与扩展

2. 算法原理

2.1 朴素思路

  • 直接遍历所有可能的路径，计算总距离，选择最短的一条。

2.2 优化路径

4.2 算法原理

  • 朴素思路: 遍历所有可能的组合，计算每种组合的物品数量，然后选择最大的那个。这种方法的时间复杂度是 O(n^3)。
  • 优化路径: 使用动态规划来避免重复计算。首先创建一个二维数组 dp，其中 dp[i][j] 表示前 i 个物品中选取 j 个物品的最大价值。然后，通过状态转移方程更新这个数组。最后，从 dp 数组中选择最大值作为结果。这种方法的时间复杂度是 O(n^2 * m)，其中 n 是物品数量，m 是物品种类数。

  • 核心思想: 通过构建一个二维数组来存储子问题的解，避免了重复计算，从而将时间复杂度降低到 O(n^2 * m)。

4.3 数学推导

  • 关键公式:
    • 状态转移方程：dp[i][j] = max(dp[i-1][k] + value[j], dp[i-1][j])
    • 最终结果：ans = max(dp[cntA][cntB])
  • 正确性证明: 根据动态规划的性质，如果当前状态的值大于或等于其前一个状态的值，则当前状态的值就是前一个状态的值加当前物品的价值；否则，当前状态的值就是前一个状态的值加当前物品的价值。因此，ans = max(dp[cntA][cntB]) 就是正确的结果。
