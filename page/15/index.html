Manacher算法(马拉车算法)详解

发表于 2024-05-08 分类于 Algorithm

前言

在处理字符串相关问题时，最长回文子串 是一道非常经典的题目。虽然暴力求解和动态规划都能解决这个问题，但在时间复杂度上往往不尽如人意。

Manacher 算法（俗称“马拉车算法”）是一种专门用于查找字符串中最长回文子串的线性时间算法。它巧妙地利用了回文串的对称性质，将时间复杂度优化到了 $O(N)$。本文将带你深入了解这一算法的实现原理。

字符串预处理

无论是暴力解法还是 Manacher 算法，为了统一处理奇数长度和偶数长度的回文串，我们通常需要对原字符串进行预处理：在字符间插入特殊符号（如 #），并在首尾添加不同的哨兵字符（如 ^ 和 $，防止越界）。

例如，字符串 aba 处理后变为 ^#a#b#a#$，这样无论是奇数长度还是偶数长度的回文，都可以统一为中心扩展的模式。

方法一：中心扩展法

中心扩展法是一种直观的暴力解法。遍历每一个字符作为中心，向两侧扩展，直到字符不相等为止。这种方法的时间复杂度为 $O(N^2)$。

代码实现

```python
def LongestPalindromicSubstring_CenterExpand(s):

# 字符串预处理：插入特殊字符，添加哨兵
# 例如 "aba" -> "^#a#b#a#$"
sn = ["^", "#"] + list("#".join(s)) + ["#", "^"]
ans = 0

# 遍历每一个节点作为中心
for i in range(1, len(sn) - 1):
    ext = 0
    # 向两侧扩展
    while sn[i - ext - 1] == sn[i + ext + 1]:
        ext += 1
    # 记录最大扩展半径
    ans = max(ext, ans)

return ans

测试

print(LongestPalindromicSubstring_CenterExpand(“abbacabaxabacabaa”))

方法二：Manacher 算法

Manacher 算法在中心扩展法的基础上进行了优化。它维护了一个最右回文边界 r 和对应的中心点 c。当我们在遍历新位置 i 时，利用回文串的对称性，可以直接得出 i 关于 c 对称位置 j 的回文半径，从而避免重复计算。

核心变量说明

  • p 数组：记录以每个字符为中心的最大回文半径。
  • c：当前已知的最长回文串的中心。
  • r：当前已知的最长回文串的右边界。

算法流程

  1. 初始化 c=0, r=0。
  2. 遍历字符串，对于当前索引 i：
    • 如果 i 在右边界 r 内，利用对称性初始化 p[i] = min(r - i, p[2 * c - i])。
    • 如果 i 超出右边界，初始化 p[i] = 0。
  3. 尝试向外扩展 i 的回文半径。
  4. 如果扩展后的回文右边界超过了 r，则更新 c 和 r。

代码实现

```python
def LongestPalindromicSubstring_Manacher(s):

# 字符串预处理
sn = ["^", "#"] + list("#".join(s)) + ["#", "^"]

# p 数组：记录每个位置的回文半径
p = [0 for _ in range(len(sn))]

c = 0  # 当前回文中心
r = 0  # 当前回文右边界

for i in range(1, len(sn) - 1):
    # 核心优化：利用对称性减少重复计算
    # 2*c-i 是 i 关于 c 的对称点
    if i <= r:
        p[i] = min(r - i, p[2 * c - i])

    # 尝试中心扩展
    while sn[i - p[i] - 1] == sn[i + p[i] + 1]:
        p[i] += 1

    # 如果扩展后的范围超过了当前右边界，更新中心和右边界
    if p[i] + i > r:
        r = p[i] + i
        c = i

return max(p)

测试

print(LongestPalindromicSubstring_Manacher(“abbacabaxabacabaa”))

复杂度分析

  • 时间复杂度：$O(N)$。虽然代码中包含双重循环，但由于 r 只会单调递增，且内部的扩展操作只会发生在未探测的区域，因此整体遍历次数是线性的。
  • 空间复杂度：$O(N)$。需要额外的数组 p 来存储半径信息。

总结

Manacher 算法通过利用回文串的对称性，巧妙地跳过了大量不必要的比较操作，将最长回文子串问题的解法优化到了线性时间。对于追求极致性能的字符串处理场景，它是一个非常值得掌握的算法。

蓝桥杯专题：帮派弟位（树形结构统计）

发表于 2024-04-08 分类于 算法竞赛 ， 蓝桥杯

题目描述

小明在游戏中参加了一个帮派，这一天他突然想知道自己在帮派中是什么地位。但是帮派的查询系统突然坏了，目前只能知道每个人的附属关系。请问你能帮帮他重建关系网并找出他的地位吗？

给定一个正整数 $n$，代表该帮派的总人数，并且小明的序号是 $m$。给出这 $n$ 个人中每个人的附属关系，确保给出的关系网为一棵树。

帮派地位的定义：按照自己手下有多少帮众决定的，注意手下的手下也算是自己的手下。如果手下的帮众相同，则按序号较小的在前面。

你能帮助小明找到自己的帮派地位吗？

输入输出格式

输入格式

第一行，两个正整数 $n(1 < n < 10^5)$ 和 $m(1 \le m \le n)$，代表该帮派的总人数以及小明的序号。
接下来 $n-1$ 行，每行两个正整数，格式如下：

  • $L, r (1 \le L, r \le n)$：代表序号为 $L$ 的人附属于序号为 $r$ 的人。

输出格式

一行，包含 1 个正整数，输出按手下人数多少排序后小明的排名。

解题思路

本题本质上是一个树形结构统计问题。

  1. 建图：根据输入的附属关系，构建一棵树。其中 $r$ 是父节点，$L$ 是子节点。
  2. 统计子树大小：题目中“手下的手下也算自己的手下”，这正好对应树结构中节点的子树大小（包含自身）。我们可以通过 DFS（深度优先搜索） 或递归的方式，自底向上计算每个节点的子树节点总数。
    • 对于叶子节点，手下人数为 0（或算上自己为 1，取决于排名定义是否包含自身。根据题目“手下”的定义，通常不包含自己，但需结合样例确认。此处代码逻辑按“手下人数”即子节点总数计算）。
  3. 排序：将所有节点按统计出的“手下人数”降序排列；若人数相同，按编号升序排列。
  4. 查询排名：遍历排序后的列表，找到小明 ($m$) 所在的位置并输出。

注意：原题数据范围 $N$ 达到 $10^5$，如果使用模拟链表合并的方式（如原文代码思路），时间复杂度可能达到 $O(N^2)$，会导致超时。推荐使用 $O(N)$ 的 DFS 算法。

修剪灌木

发表于 2024-04-08 更新于 2026-05-14 分类于 算法竞赛

问题描述

爱丽丝要完成一项修剪灌木的工作。有 $N$ 棵灌木整齐地从左到右排成一排。爱丽丝在每天傍晚会修剪一棵灌木，让这棵灌木的高度变为 $0$ 厘米。

爱丽丝修剪灌木的顺序是从最左侧的灌木开始，每天向右修剪一棵灌木。当修剪了最右侧的灌木后，她会调转方向，下一天开始向左修剪灌木。直到修剪了最左的灌木后再次调转方向，然后如此循环往复。

灌木每天从早上到傍晚会长高 $1$ 厘米，而其余时间不会长高。在第一天的早晨，所有灌木的高度都是 $0$ 厘米。

爱丽丝想知道每棵灌木最高能长到多高？

输入格式

输入一个正整数 $N$，含义如题面所述。

输出格式

输出 $N$ 行，每行一个整数，第 $i$ 行表示从左到右第 $i$ 棵灌木最高能长到多高。

样例输入

3

样例输出

2
2
2

题目分析

本题的核心在于寻找每棵灌木在被两次修剪之间的最大时间间隔。

关键逻辑推导：

  1. 生长规律：灌木每天长高 1cm，修剪后变为 0cm。因此，一棵灌木能长到的最大高度，取决于它两次被修剪之间间隔了多少天。
  2. 修剪周期：爱丽丝的修剪路径是往返的：$1 \to N \to 1 \to N \dots$。
    • 对于第 $i$ 棵灌木，它可能在爱丽丝向右走时被修剪，也可能在向左走时被修剪。
  3. 计算最大间隔：
    • 假设爱丽丝刚修剪完第 $i$ 棵灌木（位置 $i$），此时高度归零。
    • 情况一：爱丽丝继续向右走，再折返回来修剪 $i$。
      • 她需要走到最右端 $N$，再走回来。
      • 路程距离：$(N - i) + (N - i) = 2(N - i)$。
    • 情况二：爱丽丝向左走，再折返回来修剪 $i$。
      • 她需要走到最左端 $1$，再走回来。
      • 路程距离：$(i - 1) + (i - 1) = 2(i - 1)$。
  4. 结论：
    • 每棵灌木在被再次修剪前，最大生长天数取上述两种情况的较大值。
    • 由于每天生长 1cm，最大高度即为最大天数。
    • 公式：$H_i = \max(2(i-1), 2(N-i))$

复杂度分析：
该算法仅需遍历一次 $1$ 到 $N$，时间复杂度为 $O(N)$，空间复杂度为 $O(1)$，足以通过所有测试点。
