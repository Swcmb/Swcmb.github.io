题目概览

  • 题目难度: ★★★☆☆
  • 标签分类: 算法, 数据结构
  • 核心考点: 数位排序

题意解析

  • 问题描述: 给定一个数字序列，要求将每个数字的数位进行排序。例如，输入 [12, 34, 56, 789] 应输出 [123, 345, 6789]。
  • 输入格式: 整数列表，每个整数为 n 位。
  • 输出格式: 整数列表，每个整数为 n 位。
  • 样例分析: 输入 [12, 34, 56, 789]，输出 [123, 345, 6789]。

算法思路（核心部分）

1. 问题分析

  • 数位排序的核心是找到每个数的数位顺序，并按照这个顺序重新排列。
  • 对于每个数，我们首先将其转换为字符串，然后对每个字符进行遍历，记录其在字符串中的位置。

2. 算法原理

2.1 朴素思路

  • 直接遍历每个数，逐个比较其各个数位，并按顺序排列。
  • 时间复杂度为 O(n)，空间复杂度为 O(n)。

2.2 优化路径

  • 使用前缀和数组来减少比较次数。
  • 通过预处理，计算每个数的前缀和，以便于后续快速查找。

2.3 算法选择

  • 由于需要处理大量数据，且数位排序问题相对简单，因此采用朴素方法。
  • 对比其他算法，如归并排序、快速排序等，朴素方法在时间和空间上更为合适。

3. 数学推导（如适用）

  • 假设有数 a = a_1a_2...an，其中 a_i 是第 i 个数的数位。
  • 前缀和数组 prefixSum 初始化为 0，长度为 n+1。
  • 遍历每个数 a，更新前缀和数组：prefixSum[i] = a_i + prefixSum[i-1]。
  • 最后，根据前缀和数组的顺序，将数 a 的数位重新排列。

4. 多种解法对比（如适用）

解法 时间复杂度 空间复杂度 适用场景 代码难度
解法1 O(n) O(n) 小规模数据 简单
解法2 O(n log n) O(n) 中等规模 中等

复杂度分析

  • 时间复杂度: O(n)
  • 空间复杂度: O(n)
  • 复杂度对比: 与朴素方法相比，预处理减少了时间复杂度，但空间复杂度不变。

输入输出示例

  • 样例输入: [12, 34, 56, 789]
  • 样例输出: ['123', '345', '6789']
  • 执行过程: 遍历每个数，计算前缀和，再根据前缀和数组排序。

题目概览

  • 题目来源: 洛谷
  • 题目难度: ★★★☆☆
  • 标签分类: 算法标签、数据结构标签
  • 核心考点: 数组操作、贪心算法、离散化

题意解析

我们有一个长度为 (n) 的数组 (h)，每个位置有一个高度。每次操作可以选择一个区间 ([l, r])，将这个区间内所有位置的高度变成某个值 (w)，代价为 (w)。操作需要满足以下三组条件之一：

  1. 左贴边：(l = 1,\ r < n)，且区间内所有 (hi \neq h{r+1})，此时 (w = h_{r+1})。
  2. 右贴边：(l > 1,\ r = n)，且区间内所有 (hi \neq h{l-1})，此时 (w = h_{l-1})。
  3. 两端都不贴边：(1 < l \le r < n)，且 (h{l-1} = h{r+1})，且区间内所有 (hi \neq h{l-1})，此时 (w = h{l-1} = h{r+1})。

每个位置至多被操作一次。目标：经过若干次操作后，所有位置高度相等，求最小总代价。

算法思路（核心部分）

4.1 问题分析

  • 问题的本质：我们需要找到一个高度 (X)，使得将所有高度调整为 (X) 的总代价最小。
  • 关键观察点：
    • 最终统一的高度 (X) 必定是原数组中的某个值。
    • 相邻相同高度可以压缩，减少操作次数。
  • 数据范围：原数组长度 (n) 最大为 (10^6)，压缩后的数组长度最多为 (10^5)。

4.2 算法原理

  • 朴素思路：遍历所有可能的高度 (X)，计算将所有高度调整为 (X) 的总代价，取最小值。
    • 时间复杂度：(O(n \times \max(h_i)))，其中 (\max(h_i) \le 10^6)。
    • 空间复杂度：(O(n))。
  • 优化路径：通过离散化和贪心算法，减少不必要的计算。
    • 离散化：将所有可能的高度 (X) 映射到一个较小的范围内，减少查找时间。
    • 贪心算法：每次选择当前最优的高度 (X)，使得总代价最小。

4.3 数学推导（如适用）

  • 关键公式：段数 = (cnt_X + 1 - [h[1] == X] - [h[m] == X])。
    • 其中 (cnt_X) 是高度 (X) 在压缩数组中的出现次数。
    • 通过公式直接计算段数，避免了暴力枚举所有高度。

3. 题意解析

  • 问题描述: 给定两个日期，找出这两个日期之间所有可能的生日相遇时间。
  • 输入格式: 年、月、日（YYYY MM DD）
  • 输出格式: 一个列表，包含所有可能的相遇时间（YYYY MM DD）。
  • 样例分析: 例如，如果输入是 (2025, 12, 15)，输出应为 [(2025, 12, 16), (2025, 12, 17), (2025, 12, 18)]。

4. 算法思路（核心部分）

4.1 问题分析

  • 问题的本质是找出两个日期之间的所有可能相遇时间。
  • 关键观察点：两个日期之间的天数差。
  • 数据范围对算法选择的影响：对于较小的日期范围，使用贪心策略；对于较大的范围，考虑使用动态规划。

4.2 算法原理

4.2.1 朴素思路

  • 遍历每一天，检查是否满足条件（当前日期和目标日期相同）。
  • 时间复杂度：O(n)。
  • 空间复杂度：O(1)。

4.2.2 优化路径

  • 从朴素到最优的思考过程：
    • 贪心策略：每次选择与目标日期最接近的日期，但不考虑其他日期。
    • 动态规划：构建一个数组dp，记录每个日期之前是否有符合条件的相遇时间。
  • 算法选择：动态规划。

4.2.3 核心思想

  • 用通俗易懂的语言解释算法本质：通过构建一个数组，记录每个日期之前是否有符合条件的相遇时间，从而找到所有可能的相遇时间。

4.3 数学推导（如适用）

  • 关键公式：dp[i] = dp[i-1] + dp[i+1] - dp[i-days] - dp[i+days]。
