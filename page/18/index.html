C++ 类与对象基础：封装一个数组最大值求解器

发表于 2024-03-20 分类于 编程基础

前言

在 C++ 面向对象编程（OOP）中，类是构建程序的核心基石。类不仅封装了数据，还定义了操作这些数据的方法。本文将通过一个经典的示例——封装一个求数组最大值的类 Array_max，来深入解析 C++ 类的定义、成员访问控制、成员函数实现以及对象的实例化过程。

代码实现

本示例包含两个文件：头文件 array_max.h 用于类的声明，源文件 array_max.cpp 用于主程序的逻辑调用。

1. 类定义文件 (array_max.h)

在头文件中，我们定义了 Array_max 类，包含私有数据成员（数组与最大值）和公有成员函数（设置值、计算最大值、显示结果）。

```cpp
// array_max.h
// 注意：实际工程中通常不推荐使用 ，此处保留原代码风格

include

using namespace std;

class Array_max
{
public:
// 设置数组元素的值
void set_value()
{
for(int i = 0; i < 10; i++)
{
cin >> array[i];
}
};

// 计算数组中的最大值
void max_value()
{
    max = array[0]; // 初始化最大值为首元素
    for(int i = 0; i < 10; i++)
    {
        if(array[i] > max)
        {
            max = array[i];
        }
    }
};

// 输出最大值
void show_value()
{
    cout << "max=" << max;
};

private:
int array[10]; // 私有成员：存储数组
int max; // 私有成员：存储最大值
};

2. 主程序文件 (array_max.cpp)

在主程序中，我们将类的定义包含进来，并实例化对象，通过对象调用其公有接口完成任务。

```cpp

{
// 1. 实例化对象
Array_max arrmax;

// 2. 调用成员函数进行逻辑处理
arrmax.set_value();  // 输入数据
arrmax.max_value();   // 计算最大值
arrmax.show_value();  // 输出结果

}

核心知识点解析

通过上述代码，我们可以梳理出 C++ 类设计的几个关键概念：

1. 类的定义与封装

使用关键字 class 定义了 Array_max 类。类将数据（array 和 max）与操作数据的函数捆绑在一起，这就是封装。

  • Public（公有）：set_value、max_value 和 show_value 被声明为公有成员，构成了类对外部的接口。外部代码只能通过这些接口来操作对象，而无法直接访问内部数据。
  • Private（私有）：array 和 max 被声明为私有成员。这种设计保护了数据的完整性，防止外部代码随意修改数组内容，体现了“信息隐藏”的原则。

2. 构造函数与对象初始化

在本例中，没有显式定义构造函数。因此，编译器会自动生成一个默认构造函数。

  • 需要注意的是，编译器生成的默认构造函数不会对基本数据类型（如 int）进行初始化。在实际工程中，建议显式定义构造函数以初始化成员变量，避免出现未定义行为。

3. 成员函数的设计

代码中的三个成员函数职责分明，符合单一职责原则：

  • set_value()：负责数据输入。
  • max_value()：负责核心算法逻辑（遍历比较）。
  • show_value()：负责结果输出。

这种分离使得代码逻辑清晰，易于维护。如果未来需要修改输入方式或算法，只需修改对应的函数，而不会影响其他部分。

4. 基础语法的应用

  • 循环与判断：在 set_value 和 max_value 中，使用了 for 循环遍历数组，配合 if 条件语句实现最大值的动态更新。这是 C++ 流程控制的基础应用。
  • 输入输出流：使用标准库 cin 和 cout 进行 I/O 操作，通过引入命名空间 std 简化了代码编写。

总结

这个简单的示例展示了 C++ 面向对象编程的基础流程：定义类 -> 实例化对象 -> 调用方法。掌握好访问权限的控制、成员变量的管理以及接口的设计，是进阶 C++ 高级特性的必经之路。

蓝桥杯：统计数字

发表于 2024-03-17 更新于 2026-05-14 分类于 算法题解 ， 蓝桥杯

题目描述

题目要求：
输入 $N$ 个数字，统计每个数字出现的次数，并按照数字大小从小到大输出数字及其出现的次数。

输入格式：
第一行包含一个整数 $N$，表示数字的个数。
接下来 $N$ 行，每行包含一个整数。

输出格式：
按照数字从小到大的顺序输出，每行包含两个整数，分别为数字和该数字出现的次数。

解题思路

本题是一道典型的哈希表（Hash Table）应用题，主要考察数据统计与排序。

  1. 数据统计：使用字典来记录每个数字出现的频次。键存储数字，值存储出现的次数。
  2. 遍历输入：读取 $N$ 个数字，利用字典的查找特性，每次遇到数字时更新其计数。
  3. 排序输出：由于字典本身是无序的（在 Python 3.7+ 虽然保持插入顺序，但不符合题目要求的数值大小顺序），我们需要将字典转换为列表，并按键进行升序排序，最后格式化输出。

代码实现

```python

读取数字个数

n = int(input())

使用字典记录数字出现的频次

注意：避免使用 ‘dir’ 作为变量名，因为它与 Python 内置函数冲突

count_map = {}

for _ in range(n):
num = int(input())

# 使用 get 方法简化计数逻辑
# 如果 num 存在则返回其值加 1，否则返回默认值 0 加 1
count_map[num] = count_map.get(num, 0) + 1

将字典项转换为列表，并按键（数字大小）进行升序排序

items() 返回的是 (key, value) 元组，key=lambda x: x[0] 表示按元组的第一个元素（数字）排序

sorted_items = sorted(count_map.items(), key=lambda x: x[0])

遍历结果并输出

for num, count in sorted_items:
print(num, count)

复杂度分析

  • 时间复杂度：$O(N \log N)$。其中遍历 $N$ 个数字的时间复杂度为 $O(N)$，对字典按键排序的时间复杂度为 $O(K \log K)$，其中 $K$ 为不同数字的个数，最坏情况下 $K \approx N$，因此总复杂度由排序决定，为 $O(N \log N)$。
  • 空间复杂度：$O(N)$。需要使用字典存储所有出现的数字及其频次，最坏情况下所有数字均不相同，空间复杂度为 $O(N)$。

蓝桥杯真题：CLZ银行问题（队列模拟）

发表于 2024-03-17 更新于 2026-05-14 分类于 算法笔记

题目描述

CLZ 银行只有两个接待窗口，分别为 VIP 窗口和普通窗口。VIP 用户进入 VIP 窗口排队，剩下的用户进入普通窗口排队。

现有 $M$ 次操作，操作有以下四种类型：

  1. IN name V：表示一名叫 name 的用户到 VIP 窗口排队。
  2. OUT V：表示 VIP 窗口队头的用户离开排队。
  3. IN name N：表示一名叫 name 的用户到普通窗口排队。
  4. OUT N：表示普通窗口队头的用户离开排队。

求 $M$ 次操作结束后，VIP 窗口队列和普通窗口队列中的姓名（从头到尾输出）。

输入描述

第一行是一个整数 $M$ ($1 \le M \le 1000$)，表示一共有 $M$ 次操作。

第 2 行到第 $M+1$ 行输入操作，格式如题目描述所示。

输出描述

输出 $M$ 次操作后 VIP 窗口队列和普通窗口队列中的姓名（从头到尾），先输出 VIP 窗口队列，后输出普通窗口队列。

解题思路

本题是一道典型的队列模拟问题。

  1. 数据结构选择：
    题目明确要求“先来后到”的顺序，即“先进先出”（FIFO）。这正好符合队列的特性。我们需要维护两个队列：

    • VIP_Queue：用于存储 VIP 窗口的排队用户。
    • Normal_Queue：用于存储普通窗口的排队用户。

    在 Python 中，虽然列表可以模拟队列，但 list.pop(0) 的时间复杂度是 $O(N)$。对于数据规模较大的情况，推荐使用标准库 collections 中的 deque（双端队列），其 popleft() 操作的时间复杂度为 $O(1)$。

  2. 逻辑处理：

    • IN 操作：对应队列的“入队”操作。
      • 如果是 VIP 用户，将 name 加入 VIP_Queue。
      • 如果是普通用户，将 name 加入 Normal_Queue。
    • OUT 操作：对应队列的“出队”操作。
      • 如果是 VIP 窗口，执行 VIP_Queue.popleft()。
      • 如果是普通窗口，执行 Normal_Queue.popleft()。
      • 注意：实际编程中应考虑队列为空时出队的边界情况，但根据题目常规测试用例设定，通常默认操作合法。

  3. 输入处理技巧：
    使用 input().split() 将输入字符串分割成列表，可以方便地提取操作类型、姓名和窗口类型，避免繁琐的字符串切片索引。

代码实现

```python
from collections import deque

初始化两个队列

v_queue = deque()
n_queue = deque()

m = int(input())

for _ in range(m):

# 使用 split() 自动分割输入字符串
parts = input().split()
op = parts[0]

if op == "IN":
    name = parts[1]
    typ = parts[2]
    if typ == "V":
        v_queue.append(name)
    else:
        n_queue.append(name)
elif op == "OUT":
    typ = parts[1]
    if typ == "V":
        if v_queue:
            v_queue.popleft()
    else:
        if n_queue:
            n_queue.popleft()

输出结果

for name in v_queue:
print(name)

for name in n_queue:
print(name)

复杂度分析

  • 时间复杂度：$O(M)$。每个操作无论是入队还是出队，时间复杂度均为 $O(1)$，共 $M$ 次操作。
  • 空间复杂度：$O(M)$。最坏情况下所有操作都是入队，需要存储 $M$ 个名字。

总结

这道题主要考察对基本数据结构——队列的理解与应用。在处理此类模拟题时，清晰地拆解操作逻辑，并熟练使用 Python 标准库提供的工具（如 split 和 deque），能够极大地简化代码编写并提升程序效率。

一元三次方程求解

发表于 2024-03-17 更新于 2026-05-14 分类于 算法笔记

题目描述

给定一个形如 $ax^3 + bx^2 + cx + d = 0$ 的一元三次方程，已知各项系数（$a, b, c, d$ 均为实数）。约定该方程存在三个不同实根（根的范围在 -100 至 100 之间），且根与根之差的绝对值 $\ge 1$。

要求：由小到大依次在同一行输出这三个实根，根与根之间留有空格，结果精确到小数点后 2 位。

解题思路

本题是一道经典的数值分析问题，主要考察二分法求解方程根的应用。由于题目已经给出了明确的根的存在区间以及根之间的距离性质，我们可以利用单调性和零点存在定理来求解。

关键性质分析

  1. 连续性与零点定理：
    对于多项式函数 $f(x) = ax^3 + bx^2 + cx + d$，它在实数域上是连续的。如果在区间 $[left, right]$ 上满足 $f(left) \cdot f(right) < 0$，则该区间内必定存在一个实根。

  2. 根的分布特征：
    题目指出“根与根之差的绝对值 $\ge 1$”。这意味着在长度为 1 的相邻整数区间内（例如 $[0, 1]$、$[1, 2]$ 等），最多只可能存在一个实根。这为我们划分求解区间提供了依据。

算法流程

  1. 遍历区间：
    由于根的范围在 $[-100, 100]$ 之间，我们可以从 $-100$ 遍历到 $100$，每次检查长度为 1 的子区间 $[i, i+1]$。

  2. 判断根的存在：

    • 若 $f(i) = 0$，说明左端点恰好是一个根（根据题目条件，区间内不会同时存在两个根，且根不重合）。
    • 若 $f(i) \cdot f(i+1) < 0$，根据零点定理，说明根在区间 $(i, i+1)$ 内部。

  3. 二分逼近：
    对于存在根的区间 $(left, right)$，使用二分法不断缩小范围：

    • 取中点 $mid = (left + right) / 2$。
    • 判断 $f(mid)$ 的符号，确定根是在左半部分还是右半部分，更新区间端点。
    • 根据题目精度要求（保留 2 位小数），我们可以设定循环终止条件为区间长度 $< 0.001$，或者直接循环固定次数（如 50-60 次）以保证精度。

代码实现

```python

读取系数，注意题目说明为实数，应使用 float 类型

a, b, c, d = map(float, input().split())

def f(x):
“””定义目标函数”””
return a (x ** 3) + b (x * 2) + c x + d

遍历定义域区间 [-100, 100]

for i in range(-100, 100):
left = i
right = i + 1

y1 = f(left)
y2 = f(right)

# 情况1：左端点恰好是根
# 题目保证根之间差值>=1，故若左端点是根，右端点不可能是根，直接输出
if y1 == 0:
    print("{:.2f}".format(left), end=' ')

# 情况2：根在区间内部 (异号)
if y1 * y2 < 0:
    # 二分法求解
    # 精度控制：由于保留2位小数，循环至区间足够小即可
    while right - left >= 0.001:
        mid = (left + right) / 2
        if f(mid) * f(right) <= 0:
            # 根在 [mid, right] 区间
            left = mid
        else:
            # 根在 [left, mid] 区间
            right = mid

    # 输出右端点（此时左右端点已非常接近）
    print("{:.2f}".format(right), end=' ')

关键点总结

  1. 输入类型：题目明确系数为实数，因此 input 读取后应转换为 float 而非 int，这是容易丢分的细节。
  2. 端点处理：在遍历过程中，如果直接判断 f(i) == 0 输出，需要注意区间端点的重合问题。但由于题目保证根差值 $\ge 1$，且区间步长为 1，直接输出左端点不会导致重复输出（因为下一个区间的左端点就是当前区间的右端点，而右端点如果也是根，会在下一个循环作为左端点处理，或者通过异号条件处理）。
  3. 精度控制：二分法的循环条件通常设置为区间长度小于一个很小的阈值（如 $10^{-3}$ 或 $10^{-4}$），足以满足题目“保留两位小数”的要求。

算法笔记：约瑟夫环问题（队列模拟法）

发表于 2024-03-17 更新于 2026-05-14 分类于 算法与数据结构

问题描述

约瑟夫环是一个经典的数学应用问题。

背景：已知 $n$ 个人（以编号 $1, 2, 3 \dots n$ 分别表示）围坐在一张圆桌周围。
规则：

  1. 从编号为 $k$ 的人开始报数；
  2. 数到 $m$ 的那个人出圈；
  3. 他的下一个人又从 $1$ 开始报数，数到 $m$ 的那个人又出圈；
  4. 依此规律重复下去，直到圆桌周围剩余最后一个人，即为胜利者。

输入格式：三个整数 $n$（总人数）、$k$（起始编号）、$m$（步长）。

解题思路

本题最直观的解法是使用队列或循环列表进行模拟。

  1. 数据结构选择：使用 Python 的列表模拟圆桌队列。由于涉及到频繁的头部删除和尾部添加操作，若数据量极大建议使用 collections.deque，但在常规数据规模下，列表操作足以应对。
  2. 初始化队列：将人员编号按顺序存入列表。为了模拟从第 $k$ 个人开始，我们可以将列表进行“旋转”，即把 $k$ 之前的数据移到队尾，使得 $k$ 位于队首。
  3. 模拟过程：
    • 从队首弹出一个元素。
    • 计数器累加。
    • 如果计数器达到 $m$，该元素出局，计数器清零。
    • 如果未达到 $m$，将该元素追加到队尾（模拟圆桌循环）。
    • 重复此过程直到队列中只剩下一个元素。

代码实现

以下是基于 Python 列表模拟的完整代码：

```python

# 异常处理：防止空列表或非法输入
if n <= 0 or m <= 0:
    return

# 1. 初始化队列
# 原始序列为 [1, 2, ..., n]
# 通过切片操作将序列调整为以 k 开头：[k, k+1, ... n, 1, 2, ... k-1]
# 这样 pop(0) 时直接从第 k 个人开始处理
queue = list(range(k, n + 1)) + list(range(1, k))

# 2. 初始化计数器
current_count = 0

print("出局顺序：")
while len(queue) > 1:
    current_count += 1
    # 取出队首元素
    person = queue.pop(0)

    if current_count == m:
        # 达到步长 m，此人出局
        print(person, end=" ")
        current_count = 0  # 计数器重置
    else:
        # 未达到步长，将此人重新加入队尾（相当于绕圆桌一圈）
        queue.append(person)

print(f"\n最终胜利者：{queue[0]}")

代码详解

  1. 列表切片初始化：
    代码中 list(range(k, n + 1)) + list(range(1, k)) 是一个非常巧妙的处理。它避免了在循环中反复判断当前位置是否是起点的开销，直接在物理结构上将起始点 $k$ 调整为逻辑上的队首（索引 0）。

  2. 循环逻辑：
    pop(0) 和 append() 的组合完美模拟了“一人出列，如果没死则排到队尾”的过程。这比使用取模运算 (%) 计算下标在某些场景下更易于理解，因为它直观地展示了人员的流动。

复杂度分析

  • 时间复杂度：$O(n \times m)$。虽然模拟了每一次报数，但因为列表的 pop(0) 操作在 Python 中是 $O(n)$ 的（需要移动后续所有元素），整体效率会随 $n$ 增大而降低。
  • 空间复杂度：$O(n)$。需要一个长度为 $n$ 的列表存储人员信息。

优化提示：如果追求更高的性能或处理百万级数据，建议使用循环链表或推导数学公式 $f(n, m) = (f(n-1, m) + m) \% n$ 进行递归求解，将时间复杂度优化至 $O(n)$ 甚至 $O(\log n)$。
