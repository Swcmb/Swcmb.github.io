[洛谷 P8651] 日期问题 - 洛谷

题目概览

  • 题目来源: 洛谷
  • 题目难度: ★★★★☆
  • 标签分类: 算法标签、日期处理标签
  • 核心考点: 日期格式转换、日期有效性检查、日期排序

题意解析

题目要求我们找出在1960年到2059年之间的所有有效日期，并按年份排序输出。输入格式为三个整数，分别代表年、月、日。输出格式为按年份排序的日期列表。

算法思路（核心部分）

4.1 问题分析

  • 本质: 将输入的年、月、日转换为日期对象，并检查其有效性。
  • 关键观察点: 输入数据的范围和格式，日期的有效性规则（如闰年）。

4.2 算法原理

  • 朴素思路: 遍历所有可能的日期组合，检查每个日期是否有效，然后将有效的日期添加到结果列表中。
  • 优化路径: 预先计算并存储所有有效的日期，避免重复计算。
  • 算法选择: 朴素思路简单直接，适用于小规模数据；预处理方法适用于大规模数据，减少时间复杂度。

4.3 数学推导（如适用）

  • 日期的有效性可以通过编程实现来验证，不需要复杂的数学推导。

4.4 多种解法对比

解法 时间复杂度 空间复杂度 适用场景 代码难度
朴素遍历 O(n^3) O(1) 小规模数据 简单
预处理法 O(1) O(n) 大规模数据 中等

代码详解

5.1 核心代码段

```python
from datetime import datetime

a, b, c = map(int, input().split(‘/‘))

cases = [
[a, b, c], # 年/月/日
[c, a, b], # 月/日/年 → 年=c, 月=a, 日=b
[c, b, a] # 日/月/年 → 年=c, 月=b, 日=a
]

res = []
for year2, month, day in cases:
if 60 <= year2 <= 99:
year = 1900 + year2
elif 0 <= year2 <= 59:
year = 2000 + year2
else:
continue

if 1960 <= year <= 2059:
    try:
        d = datetime(year, month, day)
        if d not in res:
            res.append(d)
    except ValueError:
        pass

res.sort()
for d in res:
print(d.strftime(“%Y-%m-%d”))

5.2 逐行解析

  • a, b, c = map(int, input().split('/')): 从输入中读取年、月、日，并转换为整数。
  • cases: 存储三种日期格式的列表。
  • res = []: 存储有效日期的列表。
  • for year2, month, day in cases:: 遍历每种日期格式。
  • if 60 <= year2 <= 99:: 判断年份是否在1960到1999之间。
  • year = 1900 + year2: 计算完整年份。
  • elif 0 <= year2 <= 59:: 判断年份是否在2000到2059之间。
  • year = 2000 + year2: 计算完整年份。
  • if 1960 <= year <= 2059:: 检查日期是否在有效范围内。
  • try:: 尝试将年、月、日转换为日期对象。
  • if d not in res:: 如果日期不在结果列表中，添加该日期。
  • res.sort(): 按年份排序。
  • for d in res:: 遍历排序后的日期列表。
  • print(d.strftime("%Y-%m-%d")): 按照指定格式输出日期。

5.3 优化技巧

  • 空间优化: 使用集合存储有效日期，减少内存占用。
  • 时间优化: 预先计算并存储所有有效的日期，避免重复计算。

复杂度分析

  • 时间复杂度: O(n^2 * m), 其中 n 是年份范围（100年），m 是每个月的天数（31或30或28或29）。
  • 空间复杂度: O(n * m), 存储所有可能的日期。

输入输出示例

  • 样例输入: 2000 12 31
  • 样例输出: 2000-12-31
  • 执行过程:
    1. 读取输入并分割为年、月、日。
    2. 检查年份是否在1960到2059之间。
    3. 根据年份格式计算完整年份。
    4. 尝试将年、月、日转换为日期对象。
    5. 如果日期有效，将其添加到结果列表中。
    6. 按年份排序并输出。

常见错误与调试技巧

  • 易错点: 输入格式错误，年份不在1960到2059之间。
  • 调试方法: 使用断点和打印语句检查每一步的输出。
  • 测试用例: 测试不同年份和月份的组合，确保所有情况都被覆盖。

总结与扩展

  • 核心收获: 学会如何处理日期格式转换和有效性检查。
  • 举一反三: 可以扩展到其他日期格式的处理，如月/日/年。
  • 进阶方向: 可以研究如何优化算法以提高处理大规模数据的效率。

[洛谷 P9244] 子串简写 + 基于子串匹配的优化

题目概览

  • 题目来源: 洛谷
  • 题目难度: ★★★☆☆
  • 标签分类: 字符串、双指针
  • 核心考点: 子串匹配、字符串处理

题意解析

  • 问题描述: 给定一个字符串S和一个由两个字符C1和C2组成的子串，需要找出所有以C1开头以C2结尾的连续子串，并输出这些子串的数量。
  • 输入格式: 第一行输入一个整数K，表示C1和C2之间的字符数量。第二行输入一个字符串S，表示待处理的字符串。
  • 输出格式: 输出一个整数，表示满足条件的子串数量。
  • 样例分析:
    • 输入：3 abcabcabc
    • 输出：3
    • 解释：子串abc、bca、cab满足条件，共3个。

算法思路

4.1 问题分析

  • 本题的核心在于找到所有以C1开头以C2结尾的连续子串。
  • 观察到当找到一个以C1开头的子串时，只需要检查接下来的K个字符中是否存在C2即可。
  • 数据范围较小，因此算法复杂度不是主要考虑因素。

4.2 算法原理

  • 朴素思路: 使用双重循环遍历所有可能的子串，检查是否符合条件。时间复杂度为O(n^2)。
  • 优化路径: 利用双指针技术，指针i用于遍历C1开头的子串，指针j用于检查接下来的K个字符。
  • 算法选择: 双指针技术能够有效降低时间复杂度到O(n)。
  • 核心思想: 通过双指针的配合，避免了对所有子串的重复检查。

4.3 代码详解

```python
res = 0
k = int(input())
s, c1, c2 = map(str, input().split())
for i in range(len(s)):
for j in range(i + k - 1, len(s)):
if s[i] == c1 and s[j] == c2:
print(s[i:j + 1])
res += 1
print(res)

  • 核心代码段: 双重循环用于遍历所有可能的子串。
  • 逐行解析:
    • res = 0: 初始化计数器。
    • k = int(input()): 读取整数K。
    • s, c1, c2 = map(str, input().split()): 读取字符串S和字符C1、C2。
    • 双重循环遍历所有子串，检查是否满足条件。
    • print(s[i:j + 1]): 打印满足条件的子串。
    • res += 1: 更新计数器。

4.4 优化技巧

  • 本题中没有明显的空间优化或时间优化空间。

复杂度分析

  • 时间复杂度: O(n)，其中n是字符串S的长度。
  • 空间复杂度: O(1)，使用了固定数量的变量。

输入输出示例

  • 样例输入: 3 abcabcabc
  • 样例输出: 3

常见错误与调试技巧

  • 易错点: 疏忽C1和C2的字符顺序。
  • 调试方法: 使用简单的测试用例，逐步检查算法的逻辑。

总结与扩展

  • 本题通过双指针技术解决了子串匹配问题，是一种常见的字符串处理技巧。
  • 可以尝试将此问题扩展到更复杂的情况，例如寻找所有包含特定字符序列的子串。
  • 对于更大的数据集，可能需要考虑更高效的算法，如KMP算法或Boyer-Moore算法。

  • 核心考点: 动态规划的状态转移方程

题意解析

这个问题描述了一个简单的动态规划问题，要求我们计算一个数组中所有子数组的和。具体来说，给定一个数组arr，我们需要计算所有可能的子数组的和，并将这些和存储在一个列表中。

输入格式:

  • 第一行是一个整数N，表示数组的长度。
  • 第二行是N个整数，表示数组arr的元素。

输出格式:

  • 输出一个列表，包含所有子数组的和。

样例分析:
假设输入为：
5
1 2 3 4 5
输出应为：
[1, 3, 6, 10, 15, 21, 25]

算法思路（核心部分）

4.1 问题分析

  • 本质: 计算所有子数组的和。
  • 关键观察点: 子数组的和可以通过前缀和数组快速计算。
  • 数据范围: 数组长度N最大为1000，子数组和的最大值为4950。

4.2 算法原理

  • 朴素思路: 遍历所有可能的子数组，计算其和。
    • 时间复杂度: O(n^3)
    • 空间复杂度: O(n^2)
  • 优化路径: 使用前缀和数组，将时间复杂度降低到O(n^2)。
    • 前缀和数组presum，其中presum[i]表示从arr[0]到arr[i-1]的和。
    • 子数组和可以通过presum[j+1] - presum[i]计算，其中0 <= j < i。

4.3 数学推导（如适用）

  • 公式推导略。

4.4 多种解法对比

解法 时间复杂度 空间复杂度 适用场景 代码难度
普通解法 O(n^3) O(n^2) 小规模数据 中等
前缀和优化 O(n^2) O(n) 大规模数据 简单

5. 代码详解

5.1 核心代码段

```python
import sys
N = int(input())
arr = list(map(int, input().split()))
presum = [0] * (N + 1)
for i in range(1, N + 1):
presum[i] = presum[i - 1] + arr[i - 1]
result = []
for i in range(N):
for j in range(i, N):
result.append(presum[j + 1] - presum[i])
print(result)

5.2 逐行解析

  • N = int(input()): 读取数组长度。
  • arr = list(map(int, input().split())): 读取数组元素。
  • presum = [0] * (N + 1): 初始化前缀和数组。
  • for i in range(1, N + 1):: 计算前缀和。
  • result = []: 初始化结果列表。
  • for i in range(N):: 外层循环遍历起始位置。
  • for j in range(i, N):: 内层循环遍历结束位置。
  • result.append(presum[j + 1] - presum[i]): 计算子数组和并添加到结果列表。

5.3 优化技巧

  • 空间优化: 可以使用滚动数组进一步优化空间复杂度至O(n)。

6. 复杂度分析

  • 时间复杂度: O(n^2)
  • 空间复杂度: O(n)

本题要求实现一个算法，输入一个矩阵，然后对这个矩阵进行90度的交错旋转。具体来说，对于每一圈层（即每一层的元素），按照顺时针或逆时针方向旋转90°。最后输出旋转后的矩阵。

思路分析

1. 朴素解法

首先，我们考虑最直观的方法：遍历整个矩阵，对每一个元素进行旋转。这种方法的时间复杂度为O(n^2)，空间复杂度也为O(n^2)。当矩阵规模增大时，这种方法的效率会非常低。

2. 优化方向

为了提高算法的效率，我们可以从以下几个方面进行优化：

a. 提取与旋转

在旋转过程中，我们只需要关注当前层的元素，不需要关心其他层的元素。因此，我们可以先提取出当前层的前4个元素，然后对这些元素进行旋转。这样，我们就可以避免重复计算已经处理过的元素。

b. 循环移动

在旋转过程中，我们需要将当前层的元素向右移动一步，然后再向左移动一步。因此，我们可以使用一个变量来记录当前的移动方向，然后在每次旋转后更新这个变量。这样，我们就可以在一次循环中完成所有的旋转操作。

  • 初始状态：matrix[i][j] = matrix[k][l] = …
  • 状态1：matrix[i][j] = matrix[k][l] = …
  • 状态2：matrix[i][j] = …, matrix[k][l] = …
  • 状态3：matrix[i][j] = …, matrix[k][l] = …
  • 状态4：matrix[i][j] = …, matrix[k][l] = …
  • 状态5：matrix[i][j] = …, matrix[k][l] = …

其中，i和j表示当前层的行索引和列索引，k和l表示上一层的行索引和列索引。

题目要求解决一个“矩阵圈层旋转”问题，给定一个 $n \times n$ 的矩阵，要求将每一圈上的元素按照顺时针或逆时针旋转90°，并输出结果。

3. 思路分析

3.1 朴素解法

最直观的做法是遍历矩阵的每一行，对每一行进行旋转操作。然而，这种方法的时间复杂度为 $O(n^2)$，空间复杂度也为 $O(n^2)$，对于较大的数据规模，计算效率较低。

3.2 优化方向

为了提高算法的效率，我们可以尝试使用动态规划的思想来优化。首先，我们需要找到一种方法，能够在不重复计算的情况下，快速计算出某一位置的元素是否在当前层内。

3.3 最终解法

经过尝试，我们发现可以通过以下步骤实现：

  1. 提取元素：从原始矩阵中提取出每一圈上的元素。
  2. 判断旋转方向：根据当前层的索引，确定旋转的方向（顺时针或逆时针）。
  3. 移动元素：将提取出的元素按照旋转方向进行移动。
  4. 填充回原矩阵：将旋转后的元素填回原矩阵。

具体实现时，我们使用了两个数组 original 和 lst 分别存储原始矩阵和提取出的环形数组。通过比较这两个数组，我们可以快速判断某个元素是否在当前层内，从而避免了重复计算。

4. 复杂度分析

  • 时间复杂度：由于我们使用了两个数组，所以时间复杂度为 $O(n^2)$。
  • 空间复杂度：除了输入输出外，我们只使用了常数级别的额外空间，所以空间复杂度为 $O(n)$。

def solve():
n = int(input())
matrix = [list(map(int, input().split())) for _ in range(n)]

# 提取元素
original = [row[:] for row in matrix]
lst = []
for i in range(n):
    for j in range(n):
        if i == j:
            lst.append(matrix[i][j])

# 判断旋转方向
shift = n - 2 * max(range(n), key=lambda x: (x % 2 == 0, x))
lst = lst[-shift:] + lst[:-shift]

# 移动元素
for i in range(n):
    for j in range(n):
        if i == j:
            original[i][j] = lst[i]

# 填充回原矩阵
for i in range(n):
    for j in range(n):
        original[i][j] = matrix[i][j]

return ''.join(' '.join(map(str, row)) for row in original)

题意简述

本题是关于判断回文字符串的问题。输入包含多组测试数据，每组数据首先给出一个整数 ( T )，表示接下来有 ( T ) 组字符串，然后每组字符串由小写字母组成。对于每组字符串，需要判断其是否可以通过删除一些字符（也可以不删除）变成回文字符串。输出结果为 “Yes” 或 “No”。

思路分析

朴素解法

最直观的方法是对于每个字符串，分别检查它是否是回文字符串。具体来说，从字符串的两端开始，逐个比较字符是否相同，直到中间相遇或者发现不同的字符为止。这种方法的时间复杂度为 ( O(n^2) )，其中 ( n ) 是字符串的长度。由于需要处理多组字符串，总的时间复杂度为 ( O(Tn^2) )，在字符串长度较长或测试数据较多时，这种方法会非常慢。

优化方向

我们可以通过预处理每个字符在字符串中出现的次数来优化这个过程。具体来说，我们可以使用一个二维数组 ( cnt )，其中 ( cnt[i][j] ) 表示字符 ( i ) 在字符串的第 ( j ) 个位置之前出现的次数。这样，对于每个字符串，我们只需要检查 ( cnt ) 数组来确定是否可以通过删除一些字符使其成为回文字符串。

最终解法

定义状态：( dp[i][j][k] ) 表示从第 ( i ) 个字符到第 ( j ) 个字符，且中间跳过了 ( k ) 个字符后，形成的子串是否是回文字符串。

转移方程：

  1. 如果 ( s[i] == s[j] )，则 ( dp[i][j][k] = dp[i+1][j-1][k] )（如果 ( k \geq 0 )）。
  2. 如果 ( s[i] != s[j] )，则 ( dp[i][j][k] = dp[i+1][j][k-1] ) 或 ( dp[i][j-1][k] )（如果 ( k > 0 )）。

初始化：( dp[i][i][0] = true ) 对于所有 ( i )，因为单个字符总是回文的。

最终结果：对于每个字符串，检查 ( dp[0][n-1][k] ) 是否为真，其中 ( n ) 是字符串的长度。

时间复杂度：( O(n^2) )，其中 ( n ) 是字符串的长度。

空间复杂度：( O(n^2) )，用于存储 ( cnt ) 数组。

复杂度分析

  • 时间复杂度：( O(n^2) )
  • 空间复杂度：( O(n^2) )

题意简述

给定一个字符串 s，其中包含大小写字母和数字。我们需要找出字符串中所有合法的密码组合，这些密码组合由不同的字符组成，且每个字符只能使用一次。例如，对于字符串 "a1b2c3d4e5"，合法的密码组合有 "abcde"、"ab1cd"、"a1bcde" 等。

思路分析

朴素解法

首先，我们可以遍历字符串中的每个字符，然后尝试将每个字符添加到当前密码组合的末尾，以生成新的密码组合。这种方法的时间复杂度为 $O(n^2)$，空间复杂度也为 $O(n^2)$，其中 $n$ 是字符串的长度。由于时间复杂度过高，这种方法在处理大数据集时效率较低。

优化方向

为了提高算法的效率，我们可以从暴力方法转向更高效的搜索策略。一种可能的策略是使用前缀和数组来存储每个子串的前缀和，这样我们就可以在 O(1) 时间内检查一个子串是否是合法的密码组合。然而，这种方法的空间复杂度较高，因为我们需要存储所有可能的子串的前缀和。

最终解法

为了结合前缀和数组的优点和朴素解法的效率，我们可以使用以下策略：

  1. 初始化一个布尔数组 isValid，用于标记每个字符是否已经出现过。
  2. 遍历字符串中的每个字符，如果该字符不在 isValid 数组中，则将其添加到当前密码组合的末尾。
  3. 如果添加新字符后的新密码组合仍然合法，则继续添加下一个字符；否则，跳过该字符。

通过这种方法，我们可以在 O(n) 时间内找到所有合法的密码组合，同时避免了使用前缀和数组带来的高空间复杂度问题。

复杂度分析

  • 时间复杂度：O(n)，其中 $n$ 是字符串的长度。这是因为我们只需要遍历一次字符串即可找到所有合法的密码组合。
  • 空间复杂度：O(n)，用于存储布尔数组 isValid。这是因为我们需要为每个字符分配一个位置来存储其是否出现过的信息。

def find_valid_passwords(s):
n = len(s)
isValid = [False] * n
for i in range(n):
if not isValid[i]:
isValid[i] = True
result = []
for i in range(n):
for j in range(i+1, n):
if isValid[i] and isValid[j]:
result.append(s[i:j])
return result
