Python爬虫利器：Selenium 基础入门与基本使用

发表于 2024-07-26 更新于 2026-05-14 分类于 技术开发 ， 爬虫实战

前言

在进行网络爬虫开发时，我们经常会遇到动态渲染的网页。传统的 requests 库往往难以获取由 JavaScript 动态生成的数据。这时，Selenium 就成为了我们的利器。Selenium 是一个用于 Web 应用程序测试的工具，它能够模拟真实用户操作浏览器的行为，非常适合处理复杂的动态网页抓取。

本文将介绍 Selenium 最基础的使用方法：如何启动浏览器、访问目标网址以及获取页面源码。

环境准备

在运行以下代码前，请确保已安装以下环境：

  1. Python 环境
  2. Selenium 库：可通过 pip install selenium 安装。
  3. 浏览器驱动：需要下载与本地 Chrome 浏览器版本匹配的 ChromeDriver 并配置环境变量（或使用 webdriver-manager 自动管理）。

代码实现

```python
from selenium import webdriver

1. 实例化浏览器对象

这里以 Chrome 为例，如果配置正确，将自动打开浏览器窗口

browser = webdriver.Chrome()

# 2. 访问目标网站

核心知识点解析

1. webdriver.Chrome()

这是初始化浏览器驱动的过程。Selenium 支持多种浏览器，如 Chrome、Firefox、Edge 等。如果不指定路径，Selenium 会尝试在系统环境变量 PATH 中寻找驱动程序。

2. browser.get(url)

该方法用于指示浏览器导航到指定的 URL 地址。它会等待页面加载完成（onload 事件触发）后才继续执行后续代码。

3. browser.page_source

这是爬虫开发中最常用的属性之一。与 requests.get() 返回的原始 HTML 不同，page_source 返回的是浏览器执行 JavaScript 脚本渲染后的 DOM 树结构，这正是我们能够抓取动态数据的关键。

4. input() 的作用

在脚本末尾使用 input() 是调试阶段的小技巧。如果不加这一行，脚本执行完 print 后会立即结束，浏览器窗口会瞬间关闭，不利于观察页面跳转或登录状态。加上这行代码可以让程序挂起，直到用户按下回车键。

总结

本文介绍了 Selenium 的 “Hello World” 级别应用。掌握了启动浏览器、访问页面和获取源码这三个步骤，你就已经迈入了自动化测试与动态爬虫的大门。后续我们将深入探讨元素定位、模拟点击与表单填充等进阶操作。

Python爬虫基础：urllib处理GET请求与urlencode方法详解

发表于 2024-07-26 更新于 2026-05-14 分类于 Python爬虫

前言

在使用 Python 进行网络爬虫开发时，构建 GET 请求是最基础的操作之一。当我们需要向 URL 传递参数（如搜索关键词、页码等）时，直接手动拼接字符串既繁琐又容易出错，特别是当参数包含中文或特殊字符时。

urllib 库提供了 urlencode 方法，能够优雅地解决这个问题。本文将详细介绍如何利用 urllib.parse.urlencode 方法规范化地构建 GET 请求。

核心概念：为什么要用 urlencode？

在 URL 标准规范中，查询参数（Query String）需要遵循特定的格式（如 key=value&key2=value2）。如果参数值中包含空格、中文或其他非 ASCII 字符，浏览器和服务器通常无法直接识别，必须进行 URL 编码（URL Encoding）。

例如，参数 {'wd': '你好'} 如果直接拼接到 URL 中会导致乱码或请求失败，而 urlencode 会将其自动转换为 wd=%E4%BD%A0%E5%A5%BD 这样的格式，确保数据传输的正确性。

Python爬虫：urllib中Handler与Opener的基本使用

发表于 2024-07-26 更新于 2026-05-14 分类于 Python爬虫

前言

在 Python 爬虫开发中，urllib 是内置的标准库。虽然我们经常使用 urllib.request.urlopen() 来发送请求，但在处理更复杂的场景（如 Cookie 处理、代理设置、HTTPS 认证）时，仅仅使用 urlopen 往往不够灵活。

这就需要引入更底层的处理器对象和自定义打开器。本文将以访问百度首页为例，详细讲解 Handler 和 Opener 的基本使用方法。

核心概念

在代码实操之前，我们需要理解两个核心概念：

  1. Handlers（处理器）：Handler 是 urllib 中处理特定协议（如 HTTP、HTTPS、FTP）或功能（如代理、Cookie、重定向）的对象。通过不同的 Handler，我们可以定制 URL 打开的行为。
  2. Opener（打开器）：Opener 是通过 build_opener() 方法将多个 Handler 组装在一起形成的对象。它相当于一个自定义的浏览器客户端，拥有 open() 方法来发送请求。
