蓝桥杯真题解析：特殊图的最短路径问题

发表于 2024-03-19 分类于 算法笔记

题目背景

本题为填空题，只需要算出结果后，在代码中使用输出语句将所填结果输出即可。

小蓝学习了最短路径之后特别高兴，他定义了一个特别的图，希望找到图中的最短路径。小蓝的图由 2021 个结点组成，依次编号 1 至 2021。

题目描述

对于两个不同的结点 $a$ 和 $b$，边的连接规则如下：

  1. 无边情况：如果 $|a - b| > 21$，则两个结点之间没有边相连。
  2. 有边情况：如果 $|a - b| \le 21$，则两个点之间有一条长度为 $a$ 和 $b$ 的最小公倍数（LCM）的无向边相连。

示例说明：

  • 结点 1 和结点 23：$|1 - 23| = 22 > 21$，故没有边相连。
  • 结点 3 和结点 24：$|3 - 24| = 21 \le 21$，有一条无向边，长度为 $LCM(3, 24) = 24$。
  • 结点 15 和结点 25：$|15 - 25| = 10 \le 21$，有一条无向边，长度为 $LCM(15, 25) = 75$。

问题： 请计算结点 1 和结点 2021 之间的最短路径长度是多少？

提示： 建议使用计算机编程解决问题。

解题思路

本题本质上是一个单源最短路径问题，适合使用 Dijkstra 算法 求解。解题步骤主要分为两步：

1. 建图

我们需要构建一个大小为 $2021 \times 2021$ 的邻接矩阵来存储图结构。

  • 对于任意两个节点 $i$ 和 $j$：
    • 如果 $|i - j| \le 21$，边的权重为 $LCM(i, j)$。
    • 如果 $|i - j| > 21$，边的权重记为无穷大（表示不可达）。
    • 节点到自身的距离为 0。

2. 最短路径计算

利用 Dijkstra 算法核心逻辑：

  • 维护一个距离数组 distance，初始化为无穷大，起点的距离设为 0。
  • 每次从未访问的节点中选取距离起点最近的节点 $v$。
  • 利用节点 $v$ 对其邻接节点进行松弛操作，更新邻接节点的最短距离。
  • 重复上述过程直到所有节点都被访问或无法更新为止。

代码实现

以下是完整的 Python 代码实现，包含了最小公倍数计算、图的构建以及 Dijkstra 算法主体。

```python
import math

def lcm(a, b):

基于公式: LCM(a, b) = (a b) / GCD(a, b)
这里使用欧几里得算法手动实现 GCD
“””
x, y = a, b
while y:
x, y = y, x % y
return (a b) // x # 整数除法确保结果为整数

def dijkstra(graph, start_node, node_num):
“””
Dijkstra 最短路径算法实现
:param graph: 邻接矩阵，graph[u][v] 表示边权，inf 表示不可达
:param start_node: 起始节点编号
:param node_num: 节点总数
:return: 包含从起点到所有节点最短距离的列表
“””

# visited 数组：标记节点是否已找到最短路径
visited = [False for _ in range(node_num)]

# distance 数组：记录起点到各节点的当前最短距离
distance = [float('inf') for _ in range(node_num)]
distance[start_node] = 0

while True:
    v = -1
    # 1. 寻找未访问节点中距离最小的节点
    for u in range(node_num):
        if not visited[u] and (v == -1 or distance[u] < distance[v]):
            v = u

    # 如果找不到可更新的节点（所有节点已访问或剩余节点不可达），退出循环
    if v == -1:
        break

    # 2. 标记该节点已访问
    visited[v] = True

    # 3. 松弛操作：更新邻接节点的距离
    for u in range(node_num):
        # 如果节点 u 未访问且通过 v 到 u 更近，则更新
        if not visited[u] and graph[v][u] != float('inf'):
            distance[u] = min(distance[u], distance[v] + graph[v][u])

return distance

复杂度分析

  • 时间复杂度：$O(N^2)$。其中 $N=2021$。主要消耗在构建邻接矩阵 ($2021 \times 2021$) 和 Dijkstra 算法遍历上。对于填空题而言，这个计算量在 Python 中可以在几秒内完成。
  • 空间复杂度：$O(N^2)$。主要用于存储邻接矩阵。
