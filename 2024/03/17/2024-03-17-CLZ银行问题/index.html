蓝桥杯真题：CLZ银行问题（队列模拟）

发表于 2024-03-17 更新于 2026-05-14 分类于 算法笔记

题目描述

CLZ 银行只有两个接待窗口，分别为 VIP 窗口和普通窗口。VIP 用户进入 VIP 窗口排队，剩下的用户进入普通窗口排队。

现有 $M$ 次操作，操作有以下四种类型：

  1. IN name V：表示一名叫 name 的用户到 VIP 窗口排队。
  2. OUT V：表示 VIP 窗口队头的用户离开排队。
  3. IN name N：表示一名叫 name 的用户到普通窗口排队。
  4. OUT N：表示普通窗口队头的用户离开排队。

求 $M$ 次操作结束后，VIP 窗口队列和普通窗口队列中的姓名（从头到尾输出）。

输入描述

第一行是一个整数 $M$ ($1 \le M \le 1000$)，表示一共有 $M$ 次操作。

第 2 行到第 $M+1$ 行输入操作，格式如题目描述所示。

输出描述

输出 $M$ 次操作后 VIP 窗口队列和普通窗口队列中的姓名（从头到尾），先输出 VIP 窗口队列，后输出普通窗口队列。

解题思路

本题是一道典型的队列模拟问题。

  1. 数据结构选择：
    题目明确要求“先来后到”的顺序，即“先进先出”（FIFO）。这正好符合队列的特性。我们需要维护两个队列：

    • VIP_Queue：用于存储 VIP 窗口的排队用户。
    • Normal_Queue：用于存储普通窗口的排队用户。

    在 Python 中，虽然列表可以模拟队列，但 list.pop(0) 的时间复杂度是 $O(N)$。对于数据规模较大的情况，推荐使用标准库 collections 中的 deque（双端队列），其 popleft() 操作的时间复杂度为 $O(1)$。

  2. 逻辑处理：

    • IN 操作：对应队列的“入队”操作。
      • 如果是 VIP 用户，将 name 加入 VIP_Queue。
      • 如果是普通用户，将 name 加入 Normal_Queue。
    • OUT 操作：对应队列的“出队”操作。
      • 如果是 VIP 窗口，执行 VIP_Queue.popleft()。
      • 如果是普通窗口，执行 Normal_Queue.popleft()。
      • 注意：实际编程中应考虑队列为空时出队的边界情况，但根据题目常规测试用例设定，通常默认操作合法。

  3. 输入处理技巧：
    使用 input().split() 将输入字符串分割成列表，可以方便地提取操作类型、姓名和窗口类型，避免繁琐的字符串切片索引。

代码实现

```python
from collections import deque

初始化两个队列

v_queue = deque()
n_queue = deque()

m = int(input())

for _ in range(m):

# 使用 split() 自动分割输入字符串
parts = input().split()
op = parts[0]

if op == "IN":
    name = parts[1]
    typ = parts[2]
    if typ == "V":
        v_queue.append(name)
    else:
        n_queue.append(name)
elif op == "OUT":
    typ = parts[1]
    if typ == "V":
        if v_queue:
            v_queue.popleft()
    else:
        if n_queue:
            n_queue.popleft()

输出结果

for name in v_queue:
print(name)

for name in n_queue:
print(name)

复杂度分析

  • 时间复杂度：$O(M)$。每个操作无论是入队还是出队，时间复杂度均为 $O(1)$，共 $M$ 次操作。
  • 空间复杂度：$O(M)$。最坏情况下所有操作都是入队，需要存储 $M$ 个名字。

总结

这道题主要考察对基本数据结构——队列的理解与应用。在处理此类模拟题时，清晰地拆解操作逻辑，并熟练使用 Python 标准库提供的工具（如 split 和 deque），能够极大地简化代码编写并提升程序效率。
