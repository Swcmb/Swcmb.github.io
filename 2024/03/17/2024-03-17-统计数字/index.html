蓝桥杯：统计数字

发表于 2024-03-17 更新于 2026-05-14 分类于 算法题解 ， 蓝桥杯

题目描述

题目要求：
输入 $N$ 个数字，统计每个数字出现的次数，并按照数字大小从小到大输出数字及其出现的次数。

输入格式：
第一行包含一个整数 $N$，表示数字的个数。
接下来 $N$ 行，每行包含一个整数。

输出格式：
按照数字从小到大的顺序输出，每行包含两个整数，分别为数字和该数字出现的次数。

解题思路

本题是一道典型的哈希表（Hash Table）应用题，主要考察数据统计与排序。

  1. 数据统计：使用字典来记录每个数字出现的频次。键存储数字，值存储出现的次数。
  2. 遍历输入：读取 $N$ 个数字，利用字典的查找特性，每次遇到数字时更新其计数。
  3. 排序输出：由于字典本身是无序的（在 Python 3.7+ 虽然保持插入顺序，但不符合题目要求的数值大小顺序），我们需要将字典转换为列表，并按键进行升序排序，最后格式化输出。

代码实现

```python

读取数字个数

n = int(input())

使用字典记录数字出现的频次

注意：避免使用 ‘dir’ 作为变量名，因为它与 Python 内置函数冲突

count_map = {}

for _ in range(n):
num = int(input())

# 使用 get 方法简化计数逻辑
# 如果 num 存在则返回其值加 1，否则返回默认值 0 加 1
count_map[num] = count_map.get(num, 0) + 1

将字典项转换为列表，并按键（数字大小）进行升序排序

items() 返回的是 (key, value) 元组，key=lambda x: x[0] 表示按元组的第一个元素（数字）排序

sorted_items = sorted(count_map.items(), key=lambda x: x[0])

遍历结果并输出

for num, count in sorted_items:
print(num, count)

复杂度分析

  • 时间复杂度：$O(N \log N)$。其中遍历 $N$ 个数字的时间复杂度为 $O(N)$，对字典按键排序的时间复杂度为 $O(K \log K)$，其中 $K$ 为不同数字的个数，最坏情况下 $K \approx N$，因此总复杂度由排序决定，为 $O(N \log N)$。
  • 空间复杂度：$O(N)$。需要使用字典存储所有出现的数字及其频次，最坏情况下所有数字均不相同，空间复杂度为 $O(N)$。
