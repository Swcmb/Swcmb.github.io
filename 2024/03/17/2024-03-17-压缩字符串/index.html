【蓝桥杯】压缩字符串：贪心模拟与Python实现

发表于 2024-03-17 更新于 2026-05-14 分类于 算法刷题
本文详细讲解蓝桥杯竞赛中的“压缩字符串”题目，通过贪心模拟思路实现连续字符的压缩，并提供优化后的Python代码实现。

一、 题目描述

题目名称：压缩字符串

问题描述：
实现一个算法来压缩一个字符串。压缩的具体要求如下：

  1. 有效性判断：需要判断压缩能否节省空间。仅当压缩后的字符串长度严格小于原字符串长度时，才进行压缩输出。
  2. 压缩格式：将连续相同的字符替换为“字符+数字”的形式。例如，字符串 "AAABCCDDDD" 经压缩后变为 "A3BC2D4"。注意：如果字符连续出现次数为1，则不追加数字。

二、 输入与输出

输入描述

输入一行字符串，字符串长度不超过 500。

输出描述

输出一行结果：

  • 若输入的字符串可压缩（即压缩后更短），输出压缩后的字符串。
  • 若压缩后字符串长度未变短，输出 NO。

样例输入

```text
AAABCCDDDD

样例输出

```text

A3BC2D4

三、 题目解析与思路

这道题属于典型的字符串模拟问题。我们需要遍历字符串，统计连续出现的字符及其数量。

核心思路

  1. 遍历统计：使用一个指针遍历字符串，维护一个计数器 count，记录当前字符连续出现的次数。
  2. 拼接逻辑：
    • 当下一个字符与当前字符不同时（或到达字符串末尾），停止计数。
    • 如果计数 count > 1，拼接 字符 + count。
    • 如果计数 count == 1，仅拼接 字符。
    • 重置计数器，继续处理下一个字符。
  3. 边界处理技巧：为了简化代码逻辑，可以在字符串末尾人为添加一个哨兵字符（例如 @），这样在遍历时可以统一处理原字符串的最后一个字符段，避免在循环结束后额外编写收尾代码。
  4. 结果判断：比较压缩后的字符串长度与原字符串长度，满足条件则输出，否则输出 "NO"。

四、 代码实现

以下是优化后的 Python 代码实现。代码修复了原示例中的 HTML 实体编码问题（如 &quot;），并遵循 PEP 8 编码规范，增加了必要的注释。

```python
def compress_string(s):

# 1. 添加哨兵字符：处理末尾字符组的统一逻辑
# 添加一个原字符串不可能包含的字符，确保最后一个字符组能被正确结算
s += "@"

result = ""
count = 1  # 计数器初始化为1（包含当前字符本身）

# 2. 遍历字符串（长度-1，因为末尾添加了哨兵）
for i in range(len(s) - 1):
    current_char = s[i]
    next_char = s[i + 1]

    if current_char == next_char:
        # 字符相同，计数增加
        count += 1
    else:
        # 字符不同，进行结算
        if count > 1:
            result += current_char + str(count)
        else:
            result += current_char

        # 重置计数器
        count = 1

return result

五、 复杂度分析

  • 时间复杂度：$O(N)$，其中 $N$ 是字符串的长度。我们只需要遍历一次字符串即可完成压缩。
  • 空间复杂度：$O(N)$，主要用于存储压缩后的结果字符串。在最坏情况下（例如 “ABC”），结果字符串长度与原字符串相当。

六、 总结

这道题考察了基本的字符串处理能力。通过引入“哨兵”字符，可以有效简化循环中的边界判断逻辑，使代码更加简洁优雅。在实际竞赛或面试中，清晰的逻辑边界处理往往是得分的关键。

注意：本题的关键点在于理解题目要求——只有当连续字符数大于1时才追加数字，且最终结果必须比原字符串短才输出。
