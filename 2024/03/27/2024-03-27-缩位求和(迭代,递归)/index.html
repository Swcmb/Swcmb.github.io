算法笔记：缩位求和的迭代与递归实现

发表于 2024-03-27 更新于 2026-05-14 分类于 算法笔记

题目描述

在电子计算机普及以前，人们经常使用一种粗略的方法来验算四则运算是否正确，这种方法类似于“弃九法”。

例如： 验算 248 x 15 = 3720

  1. 对乘数和被乘数分别逐位求和，若结果为多位数则继续逐位求和，直到变为一位数。
    • 248 -> 2+4+8=14 -> 1+4=5
    • 15 -> 1+5=6
    • 乘积验算：5 x 6 = 30 -> 3+0=3
  2. 对实际运算结果逐位求和：
    • 3720 -> 3+7+2+0=12 -> 1+2=3
  3. 若两边结果一致（均为3），说明计算正确的可能性很大。

任务： 请编写一个计算机程序，对给定的数字字符串进行反复逐位求和，直到结果为一位数。

输入输出描述

输入描述：
输入为一个数字字符串，表示一个 $n$ 位数（$n < 1000$）。

输出描述：
输出一位数，表示反复逐位求和的最终结果。

算法思路

本题的核心逻辑是将数字各位相加，如果结果大于等于10，则重复该过程。这是一个典型的重复计算问题，既可以使用迭代（循环）实现，也可以使用递归实现。

代码实现

1. 迭代写法

迭代算法通过循环结构重复执行操作，直到满足终止条件。在本题中，终止条件是数字仅剩一位。

核心逻辑：

  1. 定义一个函数 num_sum 计算当前数字的各位之和。
  2. 使用 while 循环判断当前数字位数是否大于1，若大于1则调用函数更新数值。

```python
def num_sum(a):
“””计算数字 a 的各位之和”””
a = str(a)
res = 0
for i in a:
res += int(i)
return res

读取输入

num = input().strip() # 建议直接读取字符串，避免大整数转换开销，逻辑处理时保持字符串或转int视情况而定
x = int(num) # 原逻辑转int处理

迭代求解

while len(str(x)) != 1:
x = num_sum(x)

print(x)

注： 原代码中使用 int(input()) 接收输入。由于题目保证 $n<1000$，Python 支持大整数，此处转换无碍，但在处理超大数字时，直接作为字符串操作往往更优。

2. 递归写法

递归算法将问题分解为更小的子问题。对于本题，如果数字已经是一位，直接返回；否则计算各位和并调用自身。

核心逻辑：

  1. 基准条件： 如果数字长度为1，直接返回该数字。
  2. 递归步骤： 计算各位之和，将和作为参数调用自身。

```python
def num_sum(a):
“””递归计算缩位求和”””

# 将输入转为字符串以便判断长度和遍历
s = str(a)

# 基准情况：如果只剩一位，返回结果
if len(s) == 1:
    return int(s)

# 递归步骤：计算各位和并递归调用
res = 0
for i in s:
    res += int(i)
return num_sum(res)

读取输入并调用

num = input().strip()
print(num_sum(num))

方法对比

特性 迭代法 递归法
实现方式 使用循环结构 函数调用自身
代码可读性 逻辑直观，结构清晰 代码简洁，更贴近数学定义
内存消耗 通常较低，无栈帧开销 每次调用需分配栈空间，深度过大可能栈溢出
适用场景 适用于大多数线性重复任务 适用于问题可自然分解为子问题的场景

对于本题，由于输入位数 $n < 1000$，递归深度非常有限（因为数字和衰减极快，如1000位的9，第一次求和也仅为9000），因此两种方法均非常安全且高效。
