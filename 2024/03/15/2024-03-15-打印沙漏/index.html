L1-002 打印沙漏【团体程序设计天梯赛】

发表于 2024-03-15 分类于 算法刷题

题目背景

L1-002 打印沙漏

本题要求你写个程序把给定的符号打印成沙漏的形状。例如给定17个“*”，要求按下列格式打印：

```text

*

所谓“沙漏形状”，是指每行输出奇数个符号；各行符号中心对齐；相邻两行符号数差2；符号数先从大到小顺序递减到1，再从小到大顺序递增；首尾符号数相等。

给定任意N个符号，不一定能正好组成一个沙漏。要求打印出的沙漏能用掉尽可能多的符号。

输入输出格式

输入格式

输入在一行给出1个正整数N（≤1000）和一个符号，中间以空格分隔。

输出格式

首先打印出由给定符号组成的最大的沙漏形状，最后在一行中输出剩下没用掉的符号数。

输入样例

```text
19 *

输出样例

```text

*

2

解题思路

本题的核心在于计算沙漏的最大行数以及控制输出格式。

  1. 数学规律推导：
    观察沙漏结构，我们可以将其分为上半部分（含中间）和下半部分。
    假设沙漏的最大宽度（第一行的符号数）为 $2k - 1$，则沙漏的行数为 $2k - 1$ 行。

  2. 输出控制：
    确定了行数后，通过循环控制每行输出的空格数和符号数。

    • 符号数：从最大宽度开始递减到1，再递增回最大宽度。
    • 空格数：每行的空格数 = (最大宽度 - 当前行符号数) / 2。

代码实现 (Python)

下面是基于上述思路的 Python 代码实现。代码通过构建列表来管理每行的符号数量和空格数量，逻辑清晰。

```python

读取输入：N为数量，symbol为符号

n, symbol = input().split()
n = int(n)

特殊情况处理：如果符号数不足以构成一个完整沙漏（最少需要1个）

题目保证N>=1，这里主要处理N<7的情况（无法构成标准的3行沙漏）

if n < 7:
print(symbol)
print(n - 1)
else:

# 用于存储每一行的符号数量
li = []
# 用于存储每一行前置的空格数量
space_li = []
space_num = 0

# 遍历奇数序列，计算最多能使用的符号
# 沙漏结构：上半部分 + 中间 + 下半部分
# 这里的逻辑是累积计算符号数，直到超过N
for i in range(1, n, 2):
    li.append(i)
    # sum(li)为上半部分，sum(li[1:])为下半部分（不包含顶点）
    if (sum(li) + sum(li[1:])) >= n:
        # 如果正好相等或超过，需要判断是否回退
        if (sum(li) + sum(li[1:])) != n:
            li.pop() # 超过了，回退一步

        # 构建完整的行数序列：
        # li[1:] 是下半部分的行数（倒序），li 是上半部分的行数
        li = (li[1:])[::-1] + li

        # 计算剩余符号数
        a = n - sum(li)
        break

# 计算每行的空格数
for i in li:
    if i == 1:
        # 到达中间行，空格数最大
        space_li.append(space_num)
        # 下半部分的空格数是对称的，利用已生成的列表反转
        space_li = space_li + (space_li[0:li.index(i)])[::-1]
        break
    space_li.append(space_num)
    space_num += 1

# 格式化输出
for i in range(len(space_li)):
    # 打印空格和符号
    print(space_li[i] * " ", end="")
    print(li[i] * symbol)

# 打印剩余符号数
print(a)

复杂度分析

  • 时间复杂度：$O(\sqrt{N})$。主要耗时在于寻找最大行数 $k$，由于 $k$ 与 $N$ 成平方根关系，循环次数很少。
  • 空间复杂度：$O(\sqrt{N})$。用于存储每行打印信息的列表长度与行数成正比。

总结

本题是典型的模拟题，重点在于发现沙漏结构的数学规律。通过推导符号总数与最大宽度的关系，可以避免繁琐的试探法，从而高效地解决此类图形打印问题。
