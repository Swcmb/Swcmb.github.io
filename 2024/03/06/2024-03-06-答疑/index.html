蓝桥杯专题：答疑与排队接水问题详解

本文将解析两道经典的贪心算法题目：「答疑」与「排队接水」。这两道题看似场景不同，但其核心解题思路一脉相承——通过局部最优推导全局最优。

题目一：答疑

题目来源：蓝桥云课 - 答疑

问题描述

有 $n$ 位同学同时找老师答疑。每位同学都预先估计了自己答疑的时间。老师可以安排答疑的顺序，同学们要依次进入老师办公室答疑。

一位同学答疑的过程如下：

  1. 进入办公室：编号为 $i$ 的同学需要 $s_i$ 毫秒。
  2. 提问与解答：编号为 $i$ 的同学需要 $a_i$ 毫秒。
  3. 发消息：答疑完成后，同学很高兴，会在课程群里面发一条消息，时间消耗忽略不计。
  4. 离开办公室：编号为 $i$ 的同学需要 $e_i$ 毫秒。

答疑从 0 时刻开始。老师想合理地安排答疑的顺序，使得同学们在课程群里面发消息的时刻之和最小。

输入输出格式

输入格式：
第一行包含一个整数 $n$。
接下来 $n$ 行，每行包含三个整数 $s_i, a_i, e_i$。

输出格式：
输出一个整数，表示同学们在课程群里面发消息的时刻之和的最小值。

样例输入

```text
3
10000 10000 10000
20000 50000 20000
30000 20000 30000

样例输出

```text
280000

解题思路

这是一个典型的调度问题，目标是最小化所有任务的完成时刻之和。

1. 定义状态与目标

对于任意一位同学 $i$，他在群里发消息的时刻 $T_i$ 取决于他前面所有同学完成流程的总时间。
设同学 $i$ 的处理流程总消耗时间为 $t_i = s_i + a_i + e_i$。
若同学 $i$ 排在第 $k$ 个位置，则他的发消息时刻为：

其中，$\sum t_j$ 为前面所有同学的总耗时，$s_i + a_i$ 为该同学自己进入和提问的时间（发消息在离开之前）。

2. 贪心策略推导

我们需要最小化 $\sum T_i$。
观察相邻的两个同学 $A$ 和 $B$：

  • 若 $A$ 在前，对总时间的贡献增量为 $A$ 的总耗时会影响 $B$ 的开始时间。
  • 若 $B$ 在前，对总时间的贡献增量为 $B$ 的总耗时会影响 $A$ 的开始时间。

根据排队问题的经典结论（如 Johnson 算法或交换论证法），让总耗时 $t_i = s_i + a_i + e_i$ 最短的同学优先答疑，可以让后续同学等待的时间最短，从而使整体发消息时刻之和最小。

结论：
按照每位同学的总消耗时间 $(s_i + a_i + e_i)$ 从小到大排序，依次计算发消息时刻即可。

代码实现

```python
def solve():
n = int(input())
students = []

# 读取数据并计算总时间
for _ in range(n):
    s, a, e = map(int, input().split())
    total_time = s + a + e
    msg_time = s + a  # 发消息发生在离开之前
    students.append((total_time, msg_time))

# 贪心策略：按总时间从小到大排序
students.sort(key=lambda x: x[0])

current_time = 0  # 当前累计的时间点
total_msg_sum = 0 # 所有人发消息时刻之和

for total_t, msg_t in students:
    # 当前同学开始答疑，发消息时刻 = 累计时间 + (进入+提问时间)
    total_msg_sum += current_time + msg_t
    # 更新累计时间，供下一位同学使用
    current_time += total_t

print(total_msg_sum)

if name == “main“:

solve()

题目二：排队接水

题目来源：洛谷 P1223

问题描述

有 $n$ 个人在一个水龙头前排队接水，假如每个人接水的时间为 $T_i$，请编程找出这 $n$ 个人排队的一种顺序，使得 $n$ 个人的平均等待时间最小。

输入输出格式

输入格式：
第一行为一个整数 $n$。
第二行 $n$ 个整数，第 $i$ 个整数 $T_i$ 表示第 $i$ 个人的接水时间。

输出格式：
第一行为一种平均时间最短的排队顺序（输出编号，用空格分隔）。
第二行为这种排列方案下的平均等待时间（输出结果精确到小数点后两位）。

样例输入

```text
10
56 12 1 99 1000 234 33 55 99 812

样例输出

```text
3 2 7 8 1 4 9 6 10 5
291.90

解题思路

这道题是「答疑」问题的简化版。

贪心策略

要让平均等待时间最小，就是要让总等待时间之和最小。
一个人接水的时间 $T_i$ 会影响排在他后面所有人的等待时间。

  • 如果耗时长的排在前面，后面所有人都要等很久。
  • 如果耗时短的排在前面，后面所有人等待时间就短。

结论：接水时间短的人优先接水。即按 $T_i$ 从小到大排序。

代码实现

```python
def solve_water():
n = int(input())

# 读取时间，保存原始下标（从1开始）
times = list(map(int, input().split()))
people = [(times[i], i + 1) for i in range(n)]

# 贪心策略：按接水时间从小到大排序
people.sort(key=lambda x: x[0])

# 输出排队顺序
order = [str(p[1]) for p in people]
print(' '.join(order))

# 计算总等待时间
# 注意：第一个人等待时间为0，第二个人等待时间为T1，第三个人为T1+T2...
total_wait_time = 0
current_wait = 0

# 最后一个人不需要计算等待时间，所以循环次数为 n-1
for i in range(n - 1):
    current_wait += people[i][0] # 累加当前人的接水时间作为下一个人的等待增量
    total_wait_time += current_wait

# 计算平均等待时间
avg_time = total_wait_time / n
print(f"{avg_time:.2f}")

if name == “main“:
solve_water()

总结

这两道题均属于贪心算法中的经典排序问题：

  1. 答疑：按 $(s+a+e)$ 排序，目标是让后续任务的开始时间尽可能早。
  2. 排队接水：按 $T_i$ 排序，目标是减少长耗时任务对后续任务的阻塞效应。

核心思想皆为：耗时短的任务优先执行。
