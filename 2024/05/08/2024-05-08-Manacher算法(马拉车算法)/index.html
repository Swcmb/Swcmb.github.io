Manacher算法(马拉车算法)详解

发表于 2024-05-08 分类于 Algorithm

前言

在处理字符串相关问题时，最长回文子串 是一道非常经典的题目。虽然暴力求解和动态规划都能解决这个问题，但在时间复杂度上往往不尽如人意。

Manacher 算法（俗称“马拉车算法”）是一种专门用于查找字符串中最长回文子串的线性时间算法。它巧妙地利用了回文串的对称性质，将时间复杂度优化到了 $O(N)$。本文将带你深入了解这一算法的实现原理。

字符串预处理

无论是暴力解法还是 Manacher 算法，为了统一处理奇数长度和偶数长度的回文串，我们通常需要对原字符串进行预处理：在字符间插入特殊符号（如 #），并在首尾添加不同的哨兵字符（如 ^ 和 $，防止越界）。

例如，字符串 aba 处理后变为 ^#a#b#a#$，这样无论是奇数长度还是偶数长度的回文，都可以统一为中心扩展的模式。

方法一：中心扩展法

中心扩展法是一种直观的暴力解法。遍历每一个字符作为中心，向两侧扩展，直到字符不相等为止。这种方法的时间复杂度为 $O(N^2)$。

代码实现

```python
def LongestPalindromicSubstring_CenterExpand(s):

# 字符串预处理：插入特殊字符，添加哨兵
# 例如 "aba" -> "^#a#b#a#$"
sn = ["^", "#"] + list("#".join(s)) + ["#", "^"]
ans = 0

# 遍历每一个节点作为中心
for i in range(1, len(sn) - 1):
    ext = 0
    # 向两侧扩展
    while sn[i - ext - 1] == sn[i + ext + 1]:
        ext += 1
    # 记录最大扩展半径
    ans = max(ext, ans)

return ans

测试

print(LongestPalindromicSubstring_CenterExpand(“abbacabaxabacabaa”))

方法二：Manacher 算法

Manacher 算法在中心扩展法的基础上进行了优化。它维护了一个最右回文边界 r 和对应的中心点 c。当我们在遍历新位置 i 时，利用回文串的对称性，可以直接得出 i 关于 c 对称位置 j 的回文半径，从而避免重复计算。

核心变量说明

  • p 数组：记录以每个字符为中心的最大回文半径。
  • c：当前已知的最长回文串的中心。
  • r：当前已知的最长回文串的右边界。

算法流程

  1. 初始化 c=0, r=0。
  2. 遍历字符串，对于当前索引 i：
    • 如果 i 在右边界 r 内，利用对称性初始化 p[i] = min(r - i, p[2 * c - i])。
    • 如果 i 超出右边界，初始化 p[i] = 0。
  3. 尝试向外扩展 i 的回文半径。
  4. 如果扩展后的回文右边界超过了 r，则更新 c 和 r。

代码实现

```python
def LongestPalindromicSubstring_Manacher(s):

# 字符串预处理
sn = ["^", "#"] + list("#".join(s)) + ["#", "^"]

# p 数组：记录每个位置的回文半径
p = [0 for _ in range(len(sn))]

c = 0  # 当前回文中心
r = 0  # 当前回文右边界

for i in range(1, len(sn) - 1):
    # 核心优化：利用对称性减少重复计算
    # 2*c-i 是 i 关于 c 的对称点
    if i <= r:
        p[i] = min(r - i, p[2 * c - i])

    # 尝试中心扩展
    while sn[i - p[i] - 1] == sn[i + p[i] + 1]:
        p[i] += 1

    # 如果扩展后的范围超过了当前右边界，更新中心和右边界
    if p[i] + i > r:
        r = p[i] + i
        c = i

return max(p)

测试

print(LongestPalindromicSubstring_Manacher(“abbacabaxabacabaa”))

复杂度分析

  • 时间复杂度：$O(N)$。虽然代码中包含双重循环，但由于 r 只会单调递增，且内部的扩展操作只会发生在未探测的区域，因此整体遍历次数是线性的。
  • 空间复杂度：$O(N)$。需要额外的数组 p 来存储半径信息。

总结

Manacher 算法通过利用回文串的对称性，巧妙地跳过了大量不必要的比较操作，将最长回文子串问题的解法优化到了线性时间。对于追求极致性能的字符串处理场景，它是一个非常值得掌握的算法。
