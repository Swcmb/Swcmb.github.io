Python 爬虫基础：Requests 库的基本使用

发表于 2024-07-26 分类于 Python ， Spider

前言

requests 是 Python 中最受欢迎的 HTTP 库之一，以其简洁的 API 和强大的功能著称。对于爬虫初学者来说，掌握 requests 的基本使用是迈向数据采集世界的第一步。

本文将通过一个简单的示例，详细解析 requests.get() 方法返回的 Response 对象的核心属性。

代码示例

下面是一个最基础的 GET 请求示例，我们以访问百度首页为例：

```python
import requests

目标 URL

url = ‘http://www.baidu.com‘

发送 GET 请求

response = requests.get(url)

==========================================

核心解析：一个类型和六个属性

==========================================

1. 查看响应对象的类型

print(type(response)) #

2. 设置响应的编码格式（解决中文乱码问题的关键）

response.encoding = ‘utf-8’

3. 以字符串形式返回网页源代码

适用于文本数据（如 HTML, JSON）

print(response.text)

4. 返回网页地址

如果发生重定向，这里显示的是最终的 URL

print(response.url)

5. 返回二进制网页源码

适用于下载图片、视频等二进制数据

print(response.content)

6. 返回响应的状态码

200 表示成功，404 表示未找到，500 表示服务器错误

print(response.status_code)

7. 返回响应头信息

包含服务器类型、内容类型等元信息

print(response.headers)

核心要点解析

在上述代码中，我们需要重点理解 Response 对象的一个类型和六个关键属性：

1. Response 类型

requests.get() 返回的是 requests.models.Response 类型的对象。该对象包含了服务器返回的所有信息，不仅包括网页内容，还包括状态码、头部信息等。

2. response.text vs response.content

这是初学者最容易混淆的两个属性：

  • response.text：
    • 返回的是 字符串 类型数据。
    • requests 库会根据 HTTP 头部自动推测编码并解码。
    • 用途：获取 HTML 文档、接口返回的 JSON 文本。
  • response.content：
    • 返回的是 字节 类型数据。
    • 用途：下载图片、视频、PDF 文件，或者当你需要手动指定编码进行解码时。

3. 编码问题 (encoding)

如果在打印 response.text 时出现乱码，通常是因为 requests 推测的编码不准确。
解决方法很简单，手动指定编码：
```python
response.encoding = ‘utf-8’

或者根据网页实际编码设置

response.encoding = ‘gbk’

4. 状态码 (status_code)

在编写爬虫逻辑时，第一步通常是检查状态码：

  • 200：请求成功，可以进行后续解析。
  • 403：禁止访问（可能触发了反爬机制）。
  • 404：页面不存在。
  • 500：服务器内部错误。

5. 响应头 (headers)

响应头包含了丰富的元数据，例如：

  • Content-Type: 返回内容的类型。
  • Date: 服务器时间。
  • Server: 服务器软件信息。
  • Set-Cookie: 服务器设置的 Cookie 信息。

总结

掌握 Response 对象的属性是进行复杂爬虫开发的基础。建议初学者在练习时，对比 text 和 content 的输出区别，并养成检查 status_code 的习惯。
