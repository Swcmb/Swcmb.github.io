Python爬虫基础：urllib代理的基本使用

发表于 2024-07-26 更新于 2026-05-14 分类于 Python爬虫

前言

在网络爬虫开发中，频繁请求同一网站往往会触发反爬机制，导致IP被封禁。为了解决这一问题，使用代理IP是常见的反反爬策略之一。本文将详细介绍如何利用 Python 标准库 urllib 设置代理，实现隐藏真实 IP 的请求。

核心概念

在 urllib 中使用代理，主要涉及以下三个核心对象：

  1. ProxyHandler：代理处理器，用于设置代理IP信息。
  2. build_opener：构建自定义的 opener 对象，它会将代理处理器加载进去。
  3. opener.open()：使用构建好的 opener 发送请求，此时请求将通过代理服务器转发。

代码实现

下面是一个完整的示例，演示如何通过代理访问百度搜索“ip”，并将结果保存到本地。

```python
import urllib.request

1. 设置目标URL和请求头

url = ‘http://www.baidu.com/s?wd=ip‘
headers = {
‘User-Agent’: ‘Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36’
}

2. 创建 Request 对象

req = urllib.request.Request(url=url, headers=headers)

3. 设置代理IP

注意：此处使用的代理IP为示例，实际使用时请替换为有效的代理IP

格式：{‘协议类型’: ‘IP地址:端口号’}

proxies = {
‘http’: ‘125.77.25.177:8090’
}

4. 创建 ProxyHandler 处理器

handler = urllib.request.ProxyHandler(proxies=proxies)

5. 创建自定义 opener 对象

opener = urllib.request.build_opener(handler)

6. 发送请求

此时不再使用 urllib.request.urlopen(req)，而是使用 opener.open(req)

try:
res = opener.open(req, timeout=5)

# 7. 获取响应内容
content = res.read().decode('utf-8')

# 8. 持久化存储
with open('daili.html', 'w', encoding='utf-8') as f:
    f.write(content)

print("页面抓取成功，已保存为 daili.html")

except Exception as e:
print(f”请求失败，原因：{e}”)

步骤详解

1. 构建请求对象

首先，我们需要定义目标 URL 和 User-Agent。将 User-Agent 封装进 Request 对象，可以伪装成浏览器访问，避免被简单的反爬拦截。

2. 定义代理字典

proxies 字典定义了代理服务器的协议和地址。

  • 如果是 HTTP 代理，键名为 'http'。
  • 如果是 HTTPS 代理，键名为 'https'。
  • 值的格式必须为 IP:端口。

3. 创建 Handler 和 Opener

这是最关键的一步。urllib 默认的 urlopen 方法不支持直接传入代理。我们需要：

  1. 使用 ProxyHandler 传入代理字典，生成一个处理器 handler。
  2. 使用 build_opener(handler) 构建一个自定义的 opener。这个 opener 此时已经具备了使用指定代理的能力。

4. 发送请求与保存

使用 opener.open(req) 发送请求。请求成功后，读取响应数据并将其写入 daili.html 文件。打开该文件，如果代理设置正确且有效，你应该能看到百度搜索“ip”的结果页面，且显示的 IP 地址应为代理 IP，而非你的真实 IP。

注意事项

  1. 代理有效性：代码中示例的代理 IP 可能已失效，请务必替换为你自己的可用代理。建议购买付费代理或搭建自用代理池。
  2. 超时处理：使用代理请求通常比直连慢，且容易连接失败。建议在 opener.open() 中添加 timeout 参数，并使用 try-except 捕获异常。
  3. 协议匹配：确保代理协议（http/https）与目标 URL 协议一致，或者配置支持两种协议的代理。

总结

通过 ProxyHandler 和 build_opener，我们可以灵活地在 urllib 中实现代理请求。这是爬虫进阶的重要一步，理解 Handler 和 Opener 的关系，对于后续学习 Cookie 处理、异常处理等高级功能至关重要。
