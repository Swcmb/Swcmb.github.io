Python爬虫实战：使用urllib与XPath解析百度搜索按钮

发表于 2024-07-26 更新于 2026-05-14 分类于 技术实战

前言

在 Python 爬虫开发的入门阶段，学会如何发送请求并解析数据是核心技能。本文将介绍如何利用 Python 内置库 urllib 发送带伪装头的请求，并结合 lxml 库使用 XPath 语法，精准提取百度首页“百度一下”按钮的文本内容。

核心思路

  1. 发送请求：构建带有 User-Agent 的请求头，模拟浏览器访问，避免被服务器拒绝。
  2. 获取数据：使用 urlopen 获取响应，并解码为 UTF-8 格式的字符串。
  3. 解析数据：利用 lxml.etree 将 HTML 文本转换为可解析的对象，通过 XPath 路径表达式定位目标元素。

完整代码实现

以下是本次实战的完整代码：

```python

导入所需模块

import urllib.request
from lxml import etree

1. 定义目标URL

url = ‘https://www.baidu.com‘

2. 构建请求头 (伪装浏览器)

这里的 User-Agent 模拟了 Edge 浏览器

headers = {
‘User-Agent’: ‘Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36 Edge/16.16299’
}

3. 创建请求对象并添加头部信息

request = urllib.request.Request(url=url, headers=headers)

4. 发送请求并获取响应

response = urllib.request.urlopen(request)

5. 读取响应内容并解码

百度网页通常使用 UTF-8 编码

content = response.read().decode(‘utf-8’)

6. 解析网页源码

html = etree.HTML(content)

7. 使用 XPath 提取数据

逻辑：查找所有 id=”su” 的 input 标签，并获取其 value 属性值

xpath 返回的是一个列表，通过 [0] 取出第一个元素

result = html.xpath(‘//input[@id=”su”]/@value’)[0]

8. 打印结果

print(result)

代码详解

1. 为什么要设置 Headers？

直接使用 urllib 访问百度时，Python 会发送默认的 User-Agent（通常包含 Python-urllib 字样）。百度服务器识别到这是爬虫请求，可能会拒绝服务或返回不完整的数据。因此，我们需要在请求头中添加 User-Agent，伪装成正常的浏览器访问。

2. XPath 解析逻辑

代码中的核心解析语句是：
```python
result = html.xpath(‘//input[@id=”su”]/@value’)[0]

  • //input：在整个文档中查找 input 标签。
  • [@id="su"]：筛选条件，找到 id 属性为 "su" 的节点。在百度首页，这正好对应搜索框右侧的“百度一下”按钮。
  • /@value：获取该节点的 value 属性值。

运行程序后，控制台将输出：
百度一下

总结

本案例展示了最基础的爬虫流程。虽然 urllib 是 Python 内置库，但在实际开发中，我们通常会使用更加人性化的第三方库（如 requests）来处理请求。不过，理解 urllib 的底层逻辑对于掌握 Python 网络编程大有裨益。
