Python爬虫进阶：urllib请求对象定制与UA伪装

发表于 2024-07-26 分类于 Python爬虫

前言

在使用 Python 进行网络爬虫开发时，urllib 库是我们最常接触的基础库。很多初学者在使用 urllib.request.urlopen 时会发现：直接请求往往无法获取网页数据，或者被服务器重定向到登录页/错误页。

这通常是因为我们忽略了 请求头 的设置。本文将深入讲解如何通过 urllib 定制请求对象，实现 UA 伪装，以及如何处理 URL 中的中文编码问题。

一、基础知识：URL 的组成与端口

在深入代码之前，我们需要理解 URL 的基本构成。一个完整的 URL 包含协议、主机、端口号、路径、参数和锚点。

在爬虫请求中，端口号 往往决定了协议类型：

协议类型 默认端口 说明
HTTP 80 超文本传输协议
HTTPS 443 安全套接字层超文本传输协议
MySQL 3306 数据库服务端口
Redis 6379 缓存服务端口
MongoDB 27017 数据库服务端口

我们在代码中指定的 https://www.baidu.com，浏览器实际上默认连接的是 443 端口。

二、痛点：为什么需要请求对象定制？

urlopen 方法虽然简单，但它只接受 URL 字符串或特定的 Request 对象。

如果我们直接发起请求，urllib 发送的请求头中的 User-Agent 默认是 Python-urllib/x.x。绝大多数网站会识别出这是一个脚本请求，从而通过反爬机制直接拒绝访问。

核心问题：urlopen 方法本身不支持直接传递 headers 字典参数。

解决方案：使用 urllib.request.Request 类定制请求对象，将 URL 和 Headers 封装在一起。
