Python 爬虫实战：利用 urllib 发送 POST 请求获取 KFC 门店数据

发表于 2024-07-26 更新于 2026-05-14 分类于 爬虫开发
本文详细讲解如何使用 Python 标准库 urllib 模拟 POST 请求，抓取肯德基官网门店查询数据，涵盖参数编码、请求头伪装及分页逻辑处理。

前言

在进行网络爬虫开发时，GET 请求通常用于获取静态页面，而 POST 请求则常用于提交表单数据或与 API 接口进行交互。本文将以肯德基（KFC）官网的门店查询功能为例，演示如何使用 Python 内置库 urllib 发送 POST 请求，实现数据的自动抓取。

一、需求分析

目标网站通常通过 AJAX 动态加载数据。在 KFC 官网的“餐厅查询”页面中，通过浏览器开发者工具（F12）抓包可以发现，查询接口的 URL 为：

https://www.kfc.com.cn/kfccda/ashx/GetStoreList.ashx?op=cname

该接口请求方式为 POST，且需要提交以下表单参数：

  • cname: 城市名称（例如：青岛）
  • pageIndex: 当前页码
  • pageSize: 每页显示条数

同时，为了模拟正常浏览器行为，必须在请求头中添加 User-Agent。

三、关键技术点解析

1. 数据编码

在 urllib 中，POST 请求的 data 参数必须是 bytes（字节流）类型。

  • 第一步：使用 urllib.parse.urlencode(data) 将字典转换为 key=value&key=value 格式的字符串。
  • 第二步：调用 .encode('utf-8') 将字符串转换为字节流。

2. Request 对象的构建

urllib.request.Request 类允许我们自定义请求头。如果不构建 Request 对象直接使用 urlopen，将无法方便地添加 User-Agent，导致服务器可能拒绝请求（反爬机制）。

3. 分页逻辑处理

通过循环改变 pageIndex 参数，实现了多页数据的批量获取。这是爬虫中处理分页列表的常用手段。

四、总结

本文展示了如何在不安装第三方库（如 requests）的情况下，利用 Python 标准库 urllib 完成 POST 请求的发送。虽然 urllib 的用法相对繁琐（相比 requests 库），但它是理解 HTTP 协议底层原理的重要工具。

建议读者在实际开发中，优先尝试使用 requests 库以提高效率，但同时也应掌握 urllib 的核心用法，以应对受限环境下的开发需求。
