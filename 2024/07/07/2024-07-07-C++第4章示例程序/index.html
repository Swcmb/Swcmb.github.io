C++第4章：运算符重载示例解析

发表于 2024-07-07 更新于 2026-05-14 分类于 编程语言 ， C++

本文主要梳理 C++ 中运算符重载的两种实现方式：通过成员函数重载与通过友元函数重载。我们将结合复数类（Complex）的加法运算示例，深入理解其实现原理与代码结构。

4.2 使用成员函数重载运算符

在 C++ 中，运算符重载通常以成员函数的形式实现。对于双目运算符 +，如果通过成员函数重载，运算符左侧的操作数（如 c1）就是对象本身（由 this 指针指向），右侧的操作数（如 c2）则作为参数传入。

示例代码：

```cpp

include

using namespace std;

class Complex {
public:
// 默认构造函数
Complex() {
real = 0;
imag = 0;
}

// 带参构造函数
Complex(double r, double i) {
    real = r;
    imag = i;
}

// 声明重载运算符 '+'
Complex operator+(Complex &c2);

// 输出函数
void display();

private:
double real;
double imag;
};

// 实现 ‘+’ 运算符重载
Complex Complex::operator+(Complex &c2) {
Complex c;
c.real = real + c2.real; // real 等同于 this->real
c.imag = imag + c2.imag;
return c;
}

void Complex::display() {
cout << “(“ << real << “,” << imag << “i)” << endl;
}

int main() {
Complex c1(3, 4), c2(5, -10), c3;

// 执行加法运算，实际上调用的是 c1.operator+(c2)
c3 = c1 + c2;

cout << "c1="; c1.display();
cout << "c2="; c2.display();
cout << "c1+c2="; c3.display();

return 0;

}
代码解析：

  • operator+：定义为成员函数，它能够直接访问类私有成员。
  • 调用机制：执行 c1 + c2 时，编译器将其转化为对成员函数 c1.operator+(c2) 的调用。

4.3 使用友元函数重载运算符

除了成员函数，我们还可以使用友元函数 来重载运算符。与成员函数不同，友元函数没有 this 指针，因此对于双目运算符，两个操作数都必须显式地作为参数传递。

示例代码：

```cpp

include

using namespace std;

class Complex {
public:
Complex() {
real = 0;
imag = 0;
}

Complex(double r, double i) {
    real = r;
    imag = i;
}

// 声明友元函数重载运算符 '+'
friend Complex operator+(Complex &c1, Complex &c2);

void display();

private:
double real;
double imag;
};

// 实现 ‘+’ 运算符重载（友元函数）
Complex operator+(Complex &c1, Complex &c2) {
// 友元函数不属于任何类对象，直接访问参数对象的私有成员
return Complex(c1.real + c2.real, c1.imag + c2.imag);
}

void Complex::display() {
cout << “(“ << real << “,” << imag << “i)” << endl;
}

int main() {
Complex c1(3, 4), c2(5, -10), c3;

// 执行加法运算，编译器将其转化为 operator+(c1, c2)
c3 = c1 + c2;

cout << "c1="; c1.display();
cout << "c2="; c2.display();
cout << "c1+c2="; c3.display();

return 0;

}
代码解析：

  • 参数传递：友元函数需要显式传递两个参数。
  • 适用场景：当运算符左侧的操作数不是类对象（例如 5 + c1），或者需要对标准库类进行操作时，友元函数重载显得尤为重要。
  • 对称性：友元函数重载通常能更好地处理操作数类型转换带来的对称性问题。
